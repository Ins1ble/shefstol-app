
# Требования на дизайн · SCR-003 · Карточка класса

**ID:** SCR-003 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-06-15

**Источники:** `00-foundations.md` · FR-9a · FR-30 · US-4 · UC-1

> Сквозные правила (принципы, токены, каркас, паттерн состояний, тон и общая микрокопия, доступность) описаны в `00-foundations.md` и здесь не дублируются.

---

## 1. Назначение и контекст

Экран показывает **все параметры одного кулинарного класса (слота)**, чтобы клиент мог принять решение о записи и перейти к оформлению. Это промежуточный шаг между списком классов (SCR-002) и оформлением записи (SCR-004) — вложенный экран без таб-бара (см. `00-foundations.md` §4.2).

Контекст использования — кухня/студия, яркое освещение, спешка перед началом класса (`00-foundations.md` §1): клиенту нужно быстро считать ключевые параметры (когда, какая программа, кто ведёт, сколько стоит, есть ли места) и решить, записываться ли.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** убедиться, что класс подходит (время, программа, шеф, цена), и понять, есть ли свободные места, чтобы записаться.
- **Задача экрана:** дать полную и честную картину класса за один экран и предоставить один очевидный следующий шаг — «Записаться» — либо понятно объяснить, почему запись сейчас невозможна.
- **User story:** US-4 — «Как клиент, я хочу видеть все детали класса перед записью, чтобы выбрать подходящий слот».

---

## 3. Навигация

**Входящие:**
- Тап по карточке класса в списке SCR-002.

**Исходящие:**
- **«Записаться»** (основной CTA) → SCR-004 Оформление записи. Активен только при наличии свободных мест (см. §6, §8).
- **Тап по адресу студии / «Открыть карту»** → шторка BS-004 «Адрес студии» (текстовый адрес + опциональная карта, см. `00-foundations.md` §4.5).
- **Назад** (в хедере / системный жест) → возврат к списку SCR-002.

## 4. Контент-инвентарь

Все значения приходят из данных класса/программы/шефа (read-only API). Лимиты и числа **не хардкодятся** — подставляются из данных (см. `00-foundations.md` §6).

| # | Поле (модель данных) | Что показываем | Обяз. | Примечание |
|---|----------------------|----------------|-------|------------|
| 1 | `slot.start_at` | Дата и время начала | Да | Крупно, контрастно (ключевое число). |
| 2 | `program.name` | Название программы (меню) | Да | Заголовок параметра «Программа». |
| 3 | `program.type` | Тип программы (новичковая / опытная) | Да | Бейдж/подпись рядом с названием. |
| 4 | `program.description` | Описание программы | Нет | Показывать, **если значение есть** в данных; при отсутствии блок скрывается. |
| 5 | `program.duration_min` | Длительность класса | Нет | Показывать, **если значение есть**; при отсутствии блок скрывается. |
| 6 | `chef.name` | Имя шефа | Да | Без контактов шефа (`00-foundations.md` §8.2). |
| 7 | `slot.price` | Цена за место | Да | Крупно, контрастно. |
| 8 | `slot.total_seats` / `slot.free_seats` | Всего / свободно мест | Да | Свободно — ключевое число, влияет на состояние CTA. |
| 9 | `slot.free_rental_kits` | Свободно прокатных наборов | Да | Показывается **числом** свободных прокатных наборов (из данных, без хардкода); влияет на оформление в SCR-004. |
| 10 | `slot.studio_address` | Адрес студии | Да | **Текстовый блок** (адрес из API). Тап → BS-004 (см. §6). |
| 11 | `slot.coordinates` | Координаты студии | Нет | Если API предоставляет координаты, может показываться статическая карта с пином. При отсутствии — только текстовый адрес. |

> Доступные для записи места рассчитываются по формуле `min(free_seats, program.capacity_cap, 3)`, а прокатные наборы — отдельно `rental_count ≤ free_rental_kits`. На этом экране показываем фактические `free_seats` и `free_rental_kits`, детальный расчёт лимита — на SCR-004.

## 5. Структура и иерархия

Каркас — по `00-foundations.md` §4.1: фиксированный хедер, скролл-контент, фиксированный нижний CTA. Таб-бар скрыт (вложенный экран).

- **Хедер (фикс.):** кнопка «Назад» + заголовок экрана (название программы либо «Класс»).
- **Скролл-контент:** блоки параметров класса (§4) сверху вниз по приоритету считывания:
  дата/время → программа+тип (+описание, +длительность) → **адрес студии** → шеф → места → прокатные наборы → цена.
- **Нижний CTA (фикс.):** кнопка «Записаться» во всю ширину, всегда видна, не перекрыта контентом. Рядом — строка цены/доступности (контекст решения).

```
┌─────────────────────────────────┐
│ ‹ Назад      Карточка класса     │  ← фикс. хедер
├─────────────────────────────────┤
│                                  │
│  Сб, 20 июня · 10:00             │  ← start_at (крупно)
│                                  │
│  ┌─────────────────────────────┐ │
│  │ Программа                   │ │
│  │ Итальянская паста           │ │  ← program.name
│  │ [ Новичковая ] · ~180 мин   │ │  ← program.type · duration_min (если есть)
│  │ Описание: готовим пасту      │ │  ← program.description (если есть)
│  └─────────────────────────────┘ │
│  ┌─────────────────────────────┐ │
│  │ 📍 Адрес студии              │ │  ← studio_address (текст)
│  │ Лофт на ул. Заводской, 15   │ │
│  │ [ Открыть карту › ]         │ │  ← тап → BS-004 (если есть координаты)
│  └─────────────────────────────┘ │
│  ┌─────────────────────────────┐ │
│  │ Шеф                         │ │
│  │ Анна                        │ │  ← chef.name
│  └─────────────────────────────┘ │
│  ┌─────────────────────────────┐ │
│  │ Места                       │ │
│  │ Свободно 4 из 8             │ │  ← free_seats / total_seats
│  └─────────────────────────────┘ │
│  ┌─────────────────────────────┐ │
│  │ Прокатные наборы            │ │
│  │ Свободно 5                  │ │  ← free_rental_kits (числом)
│  └─────────────────────────────┘ │
│  ┌─────────────────────────────┐ │
│  │ Цена                        │ │
│  │ 1200 ₽ за место             │ │  ← price (per-seat)
│  └─────────────────────────────┘ │
│  Оплата на месте: наличные или   │  ← напоминание (foundations §6)
│  перевод на карту.               │
│                                  │
├─────────────────────────────────┤
│        [   Записаться   ]        │  ← фикс. CTA
└─────────────────────────────────┘
```

## 6. Компоненты и поведение

| Компонент | Поведение |
|-----------|-----------|
| **Кнопка «Назад»** | Возврат на SCR-002; тач-зона ≥ минимальной (`00-foundations.md` §3.1). |
| **Блоки параметров** | Карточки/строки «лейбл + значение». Ключевые числа (дата/время, цена, свободно мест) — крупные и контрастные (`00-foundations.md` §3.2). Каждый блок — самостоятельная единица чтения. |
| **Блок «Тип программы»** | Бейдж с текстовой подписью (не только цветом — `00-foundations.md` §3.2). |
| **Блок «Описание»** | Отображается только при наличии `program.description`; при отсутствии блок скрывается (без «прочерков»). |
| **Блок «Длительность»** | Отображается только при наличии `program.duration_min`; при отсутствии блок скрывается. |
| **Блок «Адрес студии»** | **Обязательный.** Текстовый блок с адресом (`studio_address`) + опционально статичная карта при наличии координат. Тап → шторка BS-004 «Адрес студии» (текст + интерактивная карта, если применимо). Состояния и fallback — по `00-foundations.md` §4.5. |
| **Блок «Прокатные наборы»** | Показывает **число свободных прокатных наборов** из данных, без хардкода. |
| **CTA «Записаться» (enabled)** | При `free_seats > 0` — активна, переход на SCR-004. Во всю ширину, высота ≥ тач-зоны. |
| **CTA «Записаться» (disabled)** | При `free_seats = 0` — неактивна, с видимой пояснительной пометкой «Мест нет» (не только цвет/прозрачность — дублируется текстом). Тап не выполняет переход. |

## 7. Состояния экрана

Базовый паттерн — `00-foundations.md` §5. Ниже — специфика SCR-003.

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетон в форме будущего контента (блоки параметров + место под CTA). Не пустой экран (`00-foundations.md` §5). |
| **Content** | Все параметры класса (§4) + активный CTA «Записаться». Основной сценарий. |
| **Error** | Заглушка ошибки + кнопка **«Обновить»**. Текст — общий из `00-foundations.md` §6: «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |
| **Нет свободных мест** (`free_seats = 0`) | Все параметры отображены; CTA **неактивна** с пометкой «Мест нет». Спокойный тон, без давления (P6). |
| **Класс отменён / недоступен** | Параметры показаны с явной пометкой «Класс отменён» / «Слот недоступен»; CTA «Записаться» **неактивна** или скрыта. Объяснение причины нейтральным тоном. |
| **Адрес студии: загрузка / ошибка / офлайн** | По правилам `00-foundations.md` §4.5: skeleton при загрузке; при ошибке/офлайн — **fallback на текстовый адрес**. Остальные параметры и CTA «Записаться» остаются доступны. |

## 8. Валидации и микрокопия

Тон и общие тексты — `00-foundations.md` §6.

- **CTA «Записаться» неактивна при `free_seats = 0`.** Пометка рядом/на кнопке: «Мест нет». Никаких штрафов и упрёков (P6).
- **Доступность мест:** формулировка вида «Свободно `{free_seats}` из `{total_seats}`» — числа подставляются из данных класса, не хардкодятся.
- **Доступность прокатных наборов:** показывается **числом** свободных прокатных наборов (`free_rental_kits`, из данных, без хардкода); лейблы инвентаря — «Свой фартук и ножи» / «Прокатный набор» (`00-foundations.md` §6). Точный расчёт лимита применяется на SCR-004.
- **Адрес студии (обязательно):** текстовый блок с адресом (`studio_address`). Например: «Адрес студии: Лофт на ул. Заводской, 15». При наличии координат — статичная карта с пином; тап / «Открыть карту» → BS-004. Текст адреса дублирует карту (доступность, §9).
- **Цена:** показывается как цена **за одно место** (per-seat), значение `slot.price` из данных. Итоговая сумма за все места считается на SCR-004.
- **Напоминание об офлайн-оплате:** текст из `00-foundations.md` §6 — «Оплата на месте: наличные или перевод на карту.»

## 9. Доступность и mobile-first

Полные правила — `00-foundations.md` §7, §3.1, §3.2.

- Ключевые числа (дата/время начала, цена за место, свободно мест) — крупные и контрастные, читаемы на ярком свете.
- CTA «Записаться» — крупный, во всю ширину контентной области, тач-зона ≥ минимальной.
- Состояния (тип программы, «Мест нет», «отменено») не передаются только цветом — дублируются текстом/иконкой/формой.
- **Адрес студии не единственный носитель:** адрес обязательно присутствует в текстовом виде; при наличии карты у неё есть доступное имя; при недоступности карты текстовый блок остаётся (`00-foundations.md` §4.5, §7).
- Все интерактивные элементы имеют доступное имя; layout не ломается при системном увеличении шрифта.

## 10. Критерии приёмки дизайна

```gherkin
AC-001: Отображение всех параметров класса
  Дано клиент открыл карточку класса со свободными местами
  Тогда на экране показаны дата и время начала, название и тип программы,
    имя шефа, цена за место, всего и свободно мест и доступность прокатных наборов
  И описание и длительность показаны, если значения есть в данных
  И показан адрес студии (текстом) и, опционально, карта

AC-002: Переход к оформлению при наличии мест
  Дано класс имеет free_seats больше нуля
  Тогда кнопка «Записаться» активна
  Когда клиент нажимает «Записаться»
  Тогда происходит переход на экран оформления записи (SCR-004)

AC-003: CTA неактивна при отсутствии мест
  Дано класс имеет free_seats равное нулю
  Тогда кнопка «Записаться» неактивна
  И рядом с кнопкой показана пометка «Мест нет»
  И тап по кнопке не выполняет переход

AC-004: Возврат к списку
  Дано клиент находится на карточке класса
  Когда клиент нажимает «Назад»
  Тогда происходит возврат к списку классов (SCR-002)

AC-005: Состояние ошибки загрузки
  Дано запрос данных класса завершился ошибкой
  Тогда показана заглушка ошибки и кнопка «Обновить»
  Когда клиент нажимает «Обновить»
  Тогда выполняется повторный запрос данных класса

AC-006: Отменённый или недоступный класс
  Дано класс отменён или недоступен для записи
  Тогда показаны параметры класса с пометкой о причине
  И кнопка «Записаться» неактивна или скрыта

AC-007: Напоминание об офлайн-оплате
  Дано клиент видит блок цены
  Тогда показано напоминание «Оплата на месте: наличные или перевод на карту.»
```

## 11. Решения

- **Описание программы:** показывается **при наличии** `program.description`; при отсутствии блок скрывается (§4, §6).
- **Длительность класса:** показывается **при наличии** `program.duration_min`; при отсутствии блок скрывается.
- **Прокатные наборы:** показываются **числом** свободных наборов (`free_rental_kits`) (§6, §8).
- **Адрес студии:** **обязательный текстовый блок**; при наличии координат — опциональная статическая карта с пином. Тап по адресу/карте → BS-004. Компонент — `00-foundations.md` §4.5. Ключ API карт — параметр конфигурации.
- **Пищевые ограничения:** на этом экране не отображаются — вводятся на этапе бронирования (SCR-004) и видны в деталях брони (SCR-006).
