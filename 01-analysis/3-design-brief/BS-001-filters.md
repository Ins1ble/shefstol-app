**ID:** BS-001 · **Тип:** Bottom Sheet · **Зона:** АЗ · **Приоритет:** High · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-06-15

## 1. Назначение и контекст

Шторка фильтров позволяет клиенту **уточнить список классов под свой запрос** — не листать весь каталог предстоящих классов, а быстро сузить его до релевантных вариантов (нужная дата, подходящий тип программы, только со свободными местами, конкретный шеф).

Шторка вызывается с экрана SCR-002 «Список классов» и поверх него же применяет результат. Она не выполняет запрос самостоятельно для отображения — итоговая фильтрованная выдача (включая пустой результат) рендерится на SCR-002. BS-001 — это **форма выбора параметров**, а не отдельный экран результатов.

Контекст использования — кухня/студия, яркое освещение, спешка (см. `00-foundations.md` §1): выбор должен быть крупным, быстрым и выполнимым одним пальцем.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** «Хочу видеть только подходящие мне классы» — отфильтровать каталог, чтобы быстрее дойти до записи (US-3, NFR-2 «короткий путь к записи»).
- **Задача дизайна:** дать собрать комбинацию условий за минимум тапов, понятно показать, что выбрано, и вернуть на список одним действием «Применить».

---

## 3. Навигация

**Входящие переходы:**
- Тап по кнопке/иконке **«Фильтры»** на SCR-002 «Список классов» → шторка открывается снизу вверх поверх списка.

**Исходящие переходы:**
- **«Применить»** → шторка закрывается, возврат на SCR-002; список перезапрашивается/перефильтровывается с учётом выбранных условий; на SCR-002 отображается **индикатор активных фильтров**.
- **«Сбросить»** → все параметры возвращаются к значениям по умолчанию (см. §7); при последующем «Применить» SCR-002 показывает дефолтную выдачу (классы на ближайшие 7 дней, `only_available=false`).
- **Закрытие** (swipe-to-close / тап по бэкдропу / кнопка закрытия) → возврат на SCR-002 **без изменения** ранее применённых фильтров.

**Свойства bottom sheet** (бэкдроп, грабер, swipe-to-close, анимация, высота по контенту, скрытие таб-бара) — см. `00-foundations.md` §4.3.

## 4. Контент-инвентарь / Параметры фильтров

Все параметры справочные и read-only приходят из API. Логика комбинирования: **«ИЛИ» внутри группы с множественным выбором** (тип программы, шеф) и **«И» между группами** — класс попадает в выдачу, если по каждой заданной группе выполнено хотя бы одно из выбранных значений. Пример: выбраны типы «Новичковая» + «Опытная» и шефы «Анна» + «Игорь» → класс подходит, если (его тип — новичковая ИЛИ опытная) И (его шеф — Анна ИЛИ Игорь).

| # | Параметр | Поле данных | Тип значения | Поведение |
|---|----------|-------------|--------------|-----------|
| 1 | **Дата / период начала** | `start_at` | Быстрый пресет **или** произвольный диапазон (от–до) | Пресеты «Сегодня» / «Эта неделя» / «Выходные» + произвольный диапазон; по умолчанию — ближайшие 7 дней (дефолт API; больший период — выбором диапазона). |
| 2 | **Тип программы** | `program.type` | **Множественный выбор** из «новичковая» / «опытная» | Чип-лист с мультивыбором: можно выбрать оба типа (ИЛИ внутри группы). По умолчанию — не выбрано (любой). |
| 3 | **Только со свободными местами** | `free_seats > 0` | Переключатель (вкл/выкл) | По умолчанию — **выкл** (соответствует дефолту SCR-002: показываются и заполненные классы с пометкой «Мест нет»). |
| 4 | **Шеф** | `chef_id` | **Множественный выбор** из справочника шефов | Чип-лист/мульти-селект: можно выбрать нескольких (ИЛИ внутри группы). По умолчанию — не выбрано (любой); справочник грузится из API. |

> Числовые лимиты и потолки в фильтрах не зашиваются — критерий «есть места» опирается на данные класса (`free_seats`), см. `00-foundations.md` §6.

## 5. Структура и иерархия

Шторка делится на три зоны: **хедер** (грабер + заголовок + «Сбросить»), скроллируемый **контент** (группы фильтров), фиксированную **нижнюю кнопку** «Применить».

```
┌──────────────────────────────────────┐
│                ▭ грабер               │  ← swipe-to-close
├──────────────────────────────────────┤
│  Фильтры                   Сбросить ▷ │  ← хедер: заголовок + сброс
├──────────────────────────────────────┤
│  Дата начала                          │  ← группа 1 (пресеты + диапазон)
│  [Сегодня] [Эта неделя] [Выходные]    │     быстрые пресеты (один)
│  ┌──────────────┐  ┌──────────────┐   │
│  │ С: 15 июня   │  │ По: 22 июня  │   │     произвольный диапазон
│  └──────────────┘  └──────────────┘   │
│                                        │
│  Тип программы                         │  ← группа 2 (чипы, мультивыбор)
│  [✓ Новичковая]  [ Опытная ]           │
│                                        │
│  ┌──────────────────────────────────┐ │  ← группа 3 (переключатель, выкл по умолчанию)
│  │ Только со свободными местами  [ ○]│ │
│  └──────────────────────────────────┘ │
│                                        │
│  Шеф                                   │  ← группа 4 (чипы, мультивыбор)
│  [✓ Анна] [ Игорь ] [ Мария ] …       │
│         (скролл при нехватке высоты)   │
├──────────────────────────────────────┤
│ ┌────────────────────────────────────┐│  ← фикс. нижний CTA
│ │            Применить               ││
│ └────────────────────────────────────┘│
└──────────────────────────────────────┘
            ░░░ бэкдроп (затемнение) ░░░
```

Порядок групп — от наиболее частого критерия (дата, тип) к уточняющим (места, шеф).

## 6. Компоненты и поведение

### 6.1 Выбор даты (`start_at`) — пресеты + диапазон
- **Быстрые пресеты** (чипы): «Сегодня», «Эта неделя», «Выходные» — выбирается один; тап заполняет соответствующий диапазон.
- **Произвольный диапазон:** два поля «С» и «По»; тап открывает выбор даты (календарь / нативный date picker), удобный одним пальцем. Допускается выбор одной даты (С = По) — фильтр на день.
- Выбор пресета и ручной диапазон синхронизированы (ручная правка снимает пресет).
- Прошедшие даты недоступны для выбора (фильтр работает по предстоящим классам).

### 6.2 Тип программы (`program.type`) — множественный выбор
- Набор **чипов с мультивыбором**: «Новичковая» / «Опытная»; можно выбрать оба (ИЛИ внутри группы).
- Состояния чипа: невыбран / выбран (дублируется не только цветом — формой/иконкой, `00-foundations.md` §3.2).

### 6.3 Переключатель «Только со свободными местами» (`free_seats > 0`)
- Toggle с подписью; по умолчанию **выключен** (в выдаче и заполненные классы с пометкой «Мест нет»).
- Состояние считывается без опоры на цвет (положение + текст).

### 6.4 Выбор шефа (`chef_id`) — множественный выбор
- **Чип-лист / мульти-селект** шефов; можно отметить несколько (ИЛИ внутри группы). Без выбора — «любой».
- Список — справочные данные из API; на время загрузки — состояние загрузки (см. §7). Загружается отдельным справочным эндпоинтом (`GET /chefs` или эквивалент).

### 6.5 Кнопка «Применить» (фиксированная, нижняя, во всю ширину)
- Состояния: активна (всегда доступна — даже при дефолтных значениях применяет/подтверждает выбор), нажата (feedback), фокус.
- Тап → закрытие шторки + применение фильтров на SCR-002 (см. §3).

### 6.6 Кнопка/действие «Сбросить» (в хедере)
- Состояния: активно (если хоть один параметр отличается от дефолта) / неактивно или скрыто (если всё в дефолте — нечего сбрасывать).
- Тап → мгновенно возвращает все контролы к значениям по умолчанию **внутри шторки** (выдача на SCR-002 обновится после «Применить»).

### 6.7 Свойства шторки
Swipe-to-close, бэкдроп с закрытием по тапу вне, грабер, явная кнопка закрытия, анимация — берутся из `00-foundations.md` §4.3, здесь не переописываются.

## 7. Состояния экрана

Опирается на сквозной паттерн состояний из `00-foundations.md` §5.

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Фильтры по умолчанию** | Дата — без пресета и диапазона (дефолт API — ближайшие 7 дней); тип программы — ничего не выбрано (любой); «только со свободными местами» — **выкл**; шеф — ничего не выбрано (любой). «Сбросить» неактивно/скрыто. |
| **Выбраны фильтры** | Контролы отражают выбор (в т.ч. несколько типов/шефов); «Сбросить» активно; «Применить» подтверждает выбор. |
| **Загрузка списка шефов** (опц.) | Поле шефа показывает состояние загрузки (скелетон/индикатор) до получения справочника из API; остальные группы интерактивны. |
| **Ошибка загрузки справочника** (опц.) | Поле шефа показывает неблокирующую ошибку с возможностью повтора; прочие фильтры остаются доступны. |

> **Пустой результат после применения** обрабатывается **на SCR-002** как empty state (UC-3 A1/E1), а не внутри шторки — шторка к этому моменту уже закрыта. Микрокопия пустого состояния — см. §8.

## 8. Валидации и микрокопия

| Правило | Поведение / текст |
|---------|-------------------|
| **Корректность диапазона дат** | Должно выполняться «от ≤ до». При попытке выбрать «По» раньше «С» — поля не дают задать некорректный диапазон (например, выбор «С» позже «По» сдвигает «По», или показывается подсказка). Применение некорректного диапазона блокируется. |
| **Пустой результат после применения** (на SCR-002) | Подсказка изменить/сбросить фильтры: **«Нет классов по условиям. Попробуйте изменить фильтры.»** Рядом — действие вернуться к фильтрам / сбросить. |
| **Тон** | Простой, без вины и давления (`00-foundations.md` §6). |

Лейблы контролов: «Фильтры», «Дата начала» (поля «С» / «По»), «Тип программы» («Новичковая» / «Опытная»), «Только со свободными местами», «Шеф» («Любой»), «Применить», «Сбросить».

## 9. Доступность и mobile-first

Базовые требования — `00-foundations.md` §3.1, §3.2, §7. Дополнительно для BS-001:
- **Крупные контролы:** чипы типа программы, toggle и поля дат — не меньше минимальной тач-зоны (≥ 44–48 pt), с запасом отступа от соседей («мокрый палец»).
- **Выбор дат одним пальцем:** компонент даты удобен для управления большим пальцем одной руки; целевые области дней/кнопок календаря крупные.
- Состояния выбора (чип, toggle) не передаются только цветом — дублируются формой/иконкой/текстом.
- Все контролы имеют доступные имена; кнопка «Применить» достижима без скролла (фиксирована внизу).

## 10. Критерии приёмки дизайна

```gherkin
Функция: Шторка фильтров BS-001

  Сценарий: AC-001 — Открытие шторки с экрана списка
    Дано клиент на экране SCR-002 «Список классов»
    Когда он нажимает «Фильтры»
    Тогда снизу открывается шторка BS-001 с грабером, заголовком, группами фильтров и кнопкой «Применить»
    И значения отражают дефолт: даты — без пресета и диапазона (дефолт API — ближайшие 7 дней), тип — любой (ничего не выбрано), «только со свободными местами» — выкл, шеф — любой (ничего не выбрано)

  Сценарий: AC-002 — Комбинирование групп по «И», значений в группе по «ИЛИ»
    Дано открыта шторка BS-001
    Когда клиент задаёт диапазон дат, отмечает типы «Новичковая» и «Опытная» и двух шефов
    И нажимает «Применить»
    Тогда на SCR-002 остаются классы, удовлетворяющие ВСЕМ группам (AND),
      где внутри группы достаточно совпадения с любым из выбранных значений (OR)

  Сценарий: AC-003 — Применение возвращает на список с индикатором
    Дано выбран хотя бы один фильтр, отличный от дефолта
    Когда клиент нажимает «Применить»
    Тогда шторка закрывается и происходит возврат на SCR-002
    И на SCR-002 виден индикатор активных фильтров

  Сценарий: AC-004 — Сброс возвращает к дефолтной выдаче (ближайшие 7 дней)
    Дано заданы какие-либо фильтры
    Когда клиент нажимает «Сбросить» и затем «Применить»
    Тогда все параметры возвращаются к значениям по умолчанию
    И SCR-002 показывает дефолтную выдачу (классы на ближайшие 7 дней, only_available=false)
    И индикатор активных фильтров на SCR-002 снят

  Сценарий: AC-005 — Закрытие без применения сохраняет прежние фильтры
    Дано на SCR-002 ранее были применены фильтры
    Когда клиент открывает шторку, меняет значения и закрывает её свайпом вниз или тапом по бэкдропу
    Тогда ранее применённые фильтры на SCR-002 не изменяются

  Сценарий: AC-006 — Корректность диапазона дат
    Дано открыта группа «Дата начала»
    Когда клиент пытается задать дату «По» раньше даты «С»
    Тогда некорректный диапазон не применяется (поля не дают его задать либо показывают подсказку)

  Сценарий: AC-007 — Пустой результат после применения (UC-3 A1/E1)
    Дано клиент применил фильтры, под которые нет ни одного класса
    Когда происходит возврат на SCR-002
    Тогда показывается empty state с текстом «Нет классов по условиям. Попробуйте изменить фильтры.»
    И доступно действие изменить/сбросить фильтры
```

## 11. Решения и параметры конфигурации

| # | Решение |
|---|---------|
| 1 | **Дата:** быстрые пресеты «Сегодня» / «Эта неделя» / «Выходные» + произвольный диапазон (§6.1). |
| 2 | **Тип программы — множественный выбор** (чипы): можно выбрать оба типа; ИЛИ внутри группы (§6.2). |
| 3 | **Шеф — множественный выбор** (чипы/мульти-селект): можно выбрать нескольких; ИЛИ внутри группы (§6.4). |
| 4 | **Живой счётчик результатов в шторке — не делаем.** Результат и пустое состояние видны на SCR-002 после «Применить». |
| 5 | *(Параметр API, не блокер)* **Справочник шефов** грузится отдельным эндпоинтом (`GET /chefs` или эквивалент) — финализируется в API-контракте. |
