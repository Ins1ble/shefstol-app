
# Требования на дизайн · SCR-002 · Список классов

**ID:** SCR-002 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-06-15

> Экранный документ дизайн-требований. Сквозные правила (принципы, токены, каркас, состояния, микрокопия, доступность) описаны в `00-foundations.md` и **здесь не дублируются** — только ссылки и специфика экрана.

**Источники:** `00-foundations.md` · FR-9 · FR-38 · US-2 · US-3 · UC-3

---

## 1. Назначение и контекст

**SCR-002 «Список классов» — главный экран приложения** и стартовая вкладка авторизованной зоны. Показывает каталог доступных кулинарных классов и служит **единственной точкой входа в сценарий записи**.

По умолчанию отображаются классы на **ближайшие 7 дней** (дефолт API; больший период — фильтром дат), отсортированные по ближайшей дате старта и сгруппированные по дням. Заполненные классы (свободных мест нет) и отменённые студией слоты **не скрываются** — они показываются с пометкой **«Мест нет»** и неактивной кнопкой записи. Фильтр «только со свободными местами» доступен в BS-001, но **по умолчанию выключен**. Отсюда клиент открывает карточку класса для записи или уточняет выдачу через фильтры.

Экран работает с API в режиме read-only (см. `00-foundations.md` §1) и подчиняется сквозному паттерну состояний (`00-foundations.md` §5).

---

## 2. Пользователь и цель

**Роль — Клиент.** Контекст использования — кухня или студия, нередко яркое освещение, мокрые руки, спешка перед началом класса (см. `00-foundations.md` §1).

**Цель на экране:** быстро просмотреть ближайшие классы и за минимум шагов выбрать подходящий (дата, тип программы, шеф, цена, есть ли места). Список оптимизирован под **быстрый выбор**: крупные карточки, контрастные ключевые числа, минимум мелкого текста (принципы P1, P2, P4 в `00-foundations.md` §2).

## 3. Навигация

### Входящие переходы
- После успешного входа/регистрации с SCR-001 — это стартовый экран АЗ.
- По тапу на таб **«Классы»** в таб-баре (с любого корневого экрана вкладок).
- Возврат с карточки класса SCR-003 (кнопка «назад») и из сценария записи.
- Закрытие шторки фильтров BS-001 (Применить / Сбросить / закрытие).

### Исходящие переходы
- Тап по карточке класса → карточка класса SCR-003.
- Тап по кнопке/иконке **«Фильтры»** → шторка BS-001.
- Таб **«Мои записи»** → SCR-005.
- Таб **«Профиль»** → SCR-007.

Таб-бар (три раздела, подписи под иконками, поведение на вложенных экранах) описан в `00-foundations.md` §4.2 — **ссылаемся, не переописываем**.

## 4. Контент-инвентарь

Каждая **карточка класса** агрегирует данные класса, программы и шефа:

| Элемент | Источник данных | Назначение в UI |
|---------|------------------|------------------|
| Дата и время начала | `slot.start_at` | Когда класс; ключевое число (контрастно). |
| Название программы (меню) | `program.name` | Что за класс. |
| Тип программы | `program.type` (новичковая / опытная) | Подходит ли по уровню; читаемый лейбл, не только цвет. |
| Шеф | `chef.name` | Кто ведёт. |
| Цена | `slot.price` | Стоимость места; ключевое число (контрастно). |
| Всего мест | `slot.total_seats` | Размер группы (M в формулировке «N из M»). |
| Свободно мест | `slot.free_seats` | Доступность; ключевое число (контрастно). |

> **Без хардкода лимитов.** Свободные места — данные класса; по сквозному правилу (`00-foundations.md` §6) доступность мест считается как `min(free_seats, program.capacity_cap, 3)`. Прокатный фонд — **отдельная проверка** `rental_count ≤ free_rental_kits`. Конкретные числа подставляются из данных, в макет не зашиваются.

**Уровень экрана (не карточки):**
- Заголовок экрана.
- Кнопка/иконка **«Фильтры»** + **индикатор активных фильтров** (видим, когда применён хотя бы один фильтр; см. `00-foundations.md` §3.5).

## 5. Структура и иерархия

Каркас следует базовому (`00-foundations.md` §4.1): фиксированный хедер → скролл-список → таб-бар. Нижнего CTA на этом экране нет.

```
┌─────────────────────────────────┐
│ Классы          [⚙ Фильтры •]   │  ← хедер: заголовок + кнопка фильтров
│                          ▲ индикатор активных фильтров
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ [карточка класса — см. ниже] │ │
│ └─────────────────────────────┘ │  ← скролл-список карточек
│ ┌─────────────────────────────┐ │
│ │ [карточка класса]           │ │
│ └─────────────────────────────┘ │
│              ⋮ (скролл)          │
├─────────────────────────────────┤
│ [Классы] [Мои записи] [Профиль]  │  ← таб-бар (foundations §4.2)
└─────────────────────────────────┘
```

**Карточка класса (вся область — одна тап-зона):**

```
┌───────────────────────────────────────┐
│  Сб, 21 июня · 10:00                    │  ← start_at (крупно, контрастно)
│  Итальянская паста · Новичковая        │  ← program.name · program.type
│  Шеф: Анна                             │  ← chef.name
│  ─────────────────────────────────────  │
│  1500 ₽            Свободно 4 из 8      │  ← price | free_seats из total_seats
└───────────────────────────────────────┘
```

Иерархия в карточке: дата/время начала (заголовок карточки) → программа+тип → шеф → строка «цена / свободно мест». Уровни — по `00-foundations.md` §3.3. Значения-примеры в wireframe условны.

## 6. Компоненты и поведение

### 6.1 Карточка класса
- **Вся карточка — единая тап-зона** (≥ минимальной тач-зоны, `00-foundations.md` §3.1), по тапу — переход в SCR-003. Внутри карточки нет конкурирующих кликов.
- Видимая обратная связь на тап (press-state), фокус для screen reader (`00-foundations.md` §7).
- Карточки разделены явными отступами/границами (`00-foundations.md` §3.4).

### 6.2 Кнопка/иконка «Фильтры»
- Размещена в хедере; открывает BS-001.
- **Индикатор активных фильтров** — виден, когда применён ≥ 1 фильтр (точка/бейдж/счётчик); считывается с первого взгляда и дублируется не только цветом (`00-foundations.md` §3.5).
- Тач-зона ≥ минимальной; подпись/доступное имя обязательны.

### 6.3 Сортировка и группировка
- Классы сортируются **по ближайшей дате/времени начала** (раньше — выше).
- При длинном списке — **заголовки-секции по дням** (sticky-разделители «дата»), чтобы клиент ориентировался в расписании. Заголовок дня — не только цвет, но текст.

### 6.4 Обновление доступности (pull-to-refresh)
- Жест «потянуть вниз» обновляет список и **актуальную доступность мест**. Во время обновления — индикатор обновления; контент не очищается без необходимости (воспринимаемая скорость, P4).

### 6.5 Подгрузка списка
- В MVP список предстоящих классов **грузится целиком** (горизонт расписания ограничен — неделя вперёд); специальная пагинация не нужна.
- *(Задел, не блокер MVP.)* Если объём вырастет, на API закладывается **курсорная пагинация** с догрузкой по достижению конца списка и скелетоном в подвале. Включается без изменения макета.

### 6.6 Класс без свободных мест
- Класс с `free_seats = 0` **показывается** с явной пометкой **«Мест нет»** (текст + иконка/форма, не только цвет) и **некликабелен** — не открывает сценарий записи.
- Скрыть заполненные классы можно фильтром «только со свободными местами» в BS-001 (по умолчанию **выключен**).

## 7. Состояния экрана

Применяется сквозной паттерн (`00-foundations.md` §5). Специфика экрана:

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетоны карточек классов (форма будущих карточек), не пустой экран. |
| **Content** | Список карточек предстоящих классов (включая заполненные с пометкой «Мест нет»), сортировка по ближайшей дате, секции по дням. |
| **Empty (нет классов)** | Заглушка: «Пока нет доступных классов» + подсказка зайти позже / обновить. |
| **Empty (нет классов по фильтрам)** | Заглушка с подсказкой **изменить или сбросить фильтры** + быстрый доступ к BS-001 (UC-3 A1/E1). |
| **Error** | Заглушка ошибки + кнопка **«Обновить»** (общий текст сетевой ошибки — `00-foundations.md` §6). |

**Скорость отклика:** загрузка/обновление списка ощущается **< 2–3 с** (NFR-6, принцип P4); до получения данных показываются скелетоны, а не пустой экран.

## 8. Валидации и микрокопия

- **Счётчик свободных мест отражает актуальные данные класса** — обновляется по pull-to-refresh и при возврате на экран; не кэшируется в устаревшем виде.
- Формулировка доступности — **«Свободно N из M»**, где N = `free_seats`, M = `total_seats` (числа из данных, не зашиты).
- Класс без мест — пометка **«Мест нет»** (текст + иконка/форма, не только цвет).
- Лейбл типа программы — **«Новичковая» / «Опытная»** (из `program.type`).
- Цена и время — из данных класса; формат единый по приложению.
- Тексты Empty:
  - нет классов вообще — «Пока нет доступных классов. Загляните позже.»;
  - нет по фильтрам — «По выбранным фильтрам ничего не нашлось. Измените или сбросьте фильтры.»
- Тон — простой и без давления (`00-foundations.md` §6).

---

## 9. Доступность и mobile-first

- Все сквозные правила — `00-foundations.md` §7 и §3.1–3.2.
- **Крупные карточки** и единая тап-зона на карточку — удобно «мокрым пальцем» на кухне.
- **Ключевые числа контрастны и крупны**: свободно мест, цена, время начала (§3.2 foundations).
- Тип программы, индикатор фильтров и пометка «Мест нет» — не только цветом, но текстом/иконкой.
- Поддержка системного увеличения шрифта: карточка не «ломается» при крупном тексте.
- Карточки и кнопка «Фильтры» имеют доступные имена для screen reader.

## 10. Критерии приёмки дизайна

```gherkin
AC-001: По умолчанию отображаются все классы на ближайшие 7 дней
  Дано клиент авторизован и открыл вкладку «Классы»
  И фильтры не применены
  Когда список загружен
  Тогда показаны все предстоящие классы на ближайшие 7 дней,
    включая заполненные (с пометкой «Мест нет» и неактивной кнопкой)
  И каждая карточка содержит дату/время начала, программу и её тип,
    шефа, цену и «Свободно N из M»

AC-002: Переход в карточку класса по тапу
  Дано на экране есть карточка класса
  Когда клиент тапает в любую область карточки
  Тогда открывается карточка класса SCR-003 для этого класса

AC-003: Открытие фильтров и индикатор активных фильтров
  Дано клиент на экране списка классов
  Когда клиент тапает «Фильтры»
  Тогда открывается шторка BS-001
  И после применения хотя бы одного фильтра в хедере виден индикатор активных фильтров

AC-004: Пустой результат по фильтрам
  Дано применены фильтры
  Когда по ним нет ни одного класса
  Тогда показан empty state с подсказкой изменить или сбросить фильтры
  И доступен быстрый переход к фильтрам

AC-005: Обновление по pull-to-refresh
  Дано список классов отображается
  Когда клиент тянет список вниз
  Тогда список и счётчики свободных мест обновляются актуальными данными
  И во время обновления виден индикатор обновления

AC-006: Состояние загрузки — скелетоны
  Дано экран открывается и данные ещё не получены
  Когда идёт первичная загрузка
  Тогда показаны скелетоны карточек, а не пустой экран
  И отклик ощущается в пределах 2–3 секунд (NFR-6)

AC-007: Ошибка загрузки
  Дано запрос списка завершился ошибкой
  Когда отображается экран
  Тогда показана заглушка ошибки с кнопкой «Обновить»
  И повтор по «Обновить» возвращает экран в загрузку

AC-008: Класс без свободных мест
  Дано у класса нет свободных мест
  И фильтр «только со свободными местами» выключен (по умолчанию)
  Тогда класс показан в списке с пометкой «Мест нет» и некликабелен
  И при включении фильтра «только со свободными местами» такой класс скрывается

AC-009: Сортировка и группировка по дням
  Дано список содержит классы на разные даты
  Когда список загружен
  Тогда классы отсортированы по ближайшей дате/времени начала
  И при длинном списке показаны заголовки-секции по дням
```

## 11. Решения и параметры конфигурации

| # | Решение |
|---|---------|
| 1 | **Сортировка по умолчанию** — по ближайшей дате/времени начала; заголовки-секции по дням при длинном списке (§6.3). |
| 2 | **Заполненные классы** — показываются с пометкой «Мест нет» и некликабельны; скрыть можно фильтром «только со свободными» (по умолчанию выключен) (§6.6). |
| 3 | **Пагинация** — в MVP список грузится целиком (горизонт ~неделя); курсорная пагинация заложена на будущее как задел, *не блокер* (§6.5). |
