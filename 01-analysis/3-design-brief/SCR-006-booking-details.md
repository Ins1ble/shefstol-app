
# Требования на дизайн · SCR-006 · Детали брони + отмена

**ID:** SCR-006 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-06-15

> Сквозные правила (принципы, токены, каркас, паттерн состояний, доступность, тон) — в `00-foundations.md`. Документ их не дублирует, а уточняет специфику экрана.

**Источники:** `00-foundations.md` · FR-16, FR-17, FR-18, FR-46 · FR-35a · US-10 · UC-2

---

## 1. Назначение и контекст

Экран показывает полную информацию об одной записи (брони) клиента и служит точкой запуска её отмены. Это вложенный экран, открываемый из списка «Мои бронирования» (SCR-005) тапом по конкретной записи.

Две задачи экрана:

- **Просмотр.** Собрать в одном месте все параметры брони и её текущий статус, чтобы клиент понимал, на что и когда он записан, сколько мест и какой инвентарь, какие пищевые ограничения указаны, какую цену оплатить офлайн.
- **Отмена.** Дать единственное действие над записью — отменить участие. Само подтверждение и пояснение последствий (правило 2 часов) выполняется в шторке BS-003, экран лишь запускает её и отображает результат после возврата.

Контекст использования — «кухня/студия, яркое освещение, спешка» (см. `00-foundations.md` §1): статус и ключевые числа читаемы с первого взгляда, деструктивное действие защищено подтверждением.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** убедиться в деталях своей записи и при необходимости отменить её заранее, понимая последствия.
- **User story:** US-10 — как клиент, я хочу видеть детали своей брони и отменять её, чтобы освободить место, если мои планы изменились.
- **Сценарий:** UC-2 «Отмена записи» (основной поток, A1 поздняя отмена, исключения E1/E2).
- **Принцип «только свои данные»** (`00-foundations.md` P5): экран показывает данные исключительно текущего клиента.

---

## 3. Навигация

### Входящие

- Тап по карточке записи в списке «Мои бронирования» (SCR-005).

### Исходящие

- **Кнопка «Отменить»** (нижний CTA) → открывает шторку BS-003 «Подтверждение отмены». После подтверждения отмены управление возвращается на SCR-006 с обновлённым статусом.
- **Тап по адресу студии / «Открыть карту»** → шторка BS-004 «Адрес студии» (текстовый адрес + опциональная карта, см. `00-foundations.md` §4.5).
- **Назад** (в хедере) → возврат на SCR-005.

Таб-бар на этом экране скрыт — экран вложенный (см. `00-foundations.md` §4.2).

## 4. Контент-инвентарь

Источник полей — модель данных (сущность booking и связанные slot, program, chef). Конкретные имена полей сверяются с моделью при детализации.

| Блок | Поле / данные | Откуда | Примечание |
|------|---------------|--------|------------|
| Статус | `status` | booking | Активна / отменена / поздняя отмена / отменён студией / прошедшая (бейдж). |
| Когда | `slot.start_at` | slot | Дата и время начала класса. |
| Программа | `slot.program.name` + `program.type` | program | Название программы (меню) и её тип (новичковая / опытная). |
| Описание программы | `program.description` | program | Если есть. |
| Длительность | `program.duration_min` | program | Если есть. |
| Адрес студии | `slot.studio_address` (+ координаты) | slot | Обязательный. Текстовый блок. При наличии координат — опциональная карта. Тап → BS-004. |
| Шеф | `slot.chef.name` | chef | Кто ведёт класс. |
| Цена | `slot.price` / `booking.price_total` | slot / booking | Стоимость; оплата офлайн (текст — из `00-foundations.md` §6). |
| Места | `seats_count` | booking | Сколько мест в брони (себя + гости). |
| Инвентарь | вариант инвентаря по местам / `rental_count` | booking | Свой / прокатный; число прокатных наборов. Лейблы — из `00-foundations.md` §6. |
| Пищевые ограничения | `dietary_restrictions` | booking | Если заполнены — показать полностью. |
| Создана | `created_at` | booking | Когда сделана запись. |
| Отменена | `cancelled_at` | booking | Показывается только если запись отменена. |

> Числовые лимиты (потолок программы, размер фонда) в тексты не зашиваются — берутся из данных класса/программы (`00-foundations.md` §6). Лейблы «Свой фартук и ножи» / «Прокатный набор» — из той же таблицы микрокопии.

## 5. Структура и иерархия

Каркас следует `00-foundations.md` §4.1: фиксированный хедер, скролл-контент, фиксированный нижний CTA. Таб-бар отсутствует.

- **Хедер (фикс.):** кнопка «Назад» + заголовок экрана («Детали записи»).
- **Скролл-контент:** бейдж статуса (вверху, заметный) → блок «Когда» (дата/время начала) → блок «Программа и шеф» → адрес студии → блок «Места и инвентарь» → блок «Пищевые ограничения» → блок «Цена» (+ напоминание об офлайн-оплате) → служебные метки (создана / отменена).
- **Фикс. нижний CTA:** кнопка «Отменить» во всю ширину контентной области. При недоступности действия CTA показывается неактивным (disabled) с пояснением причины (см. §6, §7).

### ASCII-wireframe


## 6. Компоненты и поведение

### 6.1 Блоки деталей

Сгруппированные секции/карточки (одна колонка, `00-foundations.md` §3.4). Каждая секция имеет заголовок-лейбл и значение; ключевые числа (время начала, цена, мест) — крупные и контрастные (`00-foundations.md` §3.2). Read-only, без редактирования.

### 6.2 Бейдж статуса

Отражает `status` записи. Состояния не передаются только цветом — обязательны иконка/текст (`00-foundations.md` §3.2, §3.5). Возможные значения:

- **Активна** — предстоящая, не отменённая запись.
- **Отменена** — ранняя отмена (места и прокатные наборы освобождены).
- **Поздняя отмена** — отменена < 2 ч до начала (место не освобождено). Текст-итог — из `00-foundations.md` §6.
- **Отменён студией** — класс отменён организатором (форс-мажор). Дополнительно отображается причина отмены (`cancel_reason`), если приходит в API.
- **Прошедшая** — класс уже начался/завершён.

### 6.3 Кнопка «Отменить» (нижний CTA)

- Крупная, во всю ширину (`00-foundations.md` §3.1).
- **Enabled** — только когда запись активна И класс ещё не начался. Тап → открывает BS-003.
- **Disabled** (показывается неактивной, не скрывается) — если класс уже стартовал/прошёл (UC-2 E1) либо запись уже отменена (UC-2 E2 — повторная отмена не выполняется). Рядом — обязательное короткое пояснение причины недоступности (см. §8), чтобы причина была читаема.
- **Отмена — только целиком.** Частичная отмена отдельных мест в MVP не поддерживается (см. §11).

> Деструктивное действие не выполняется на этом экране напрямую — фактическое подтверждение, логика ранней/поздней отмены и текст правила 2 часов — в BS-003.

### 6.4 Адрес студии

- **Обязательный текстовый блок** с адресом (`studio_address`). При наличии координат — опциональная статичная карта с пином. Тап / «Открыть карту» → BS-004.
- При недоступности карты — fallback на текст (адрес + ссылка «Открыть в картах»). Состояния — по `00-foundations.md` §4.5.

## 7. Состояния экрана

Базовый паттерн — `00-foundations.md` §5. Специфика SCR-006:

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетон в форме блоков деталей и бейджа статуса (не пустой экран). |
| **Content** | Полные детали брони + статус + нижний CTA по правилам §6.3. |
| **Error** | Заглушка ошибки + кнопка **«Обновить»** (текст — `00-foundations.md` §6). |
| **Запись отменена** | Бейдж «Отменена» / «Поздняя отмена» / «Отменён студией», показан `cancelled_at`. CTA «Отменить» неактивна с пояснением «Запись уже отменена» (UC-2 E2). Для «Отменён студией» — дополнительно причина отмены. |
| **Класс начался/прошёл** | Бейдж «Прошедшая» (или «Активна» с пометкой о начале). CTA «Отменить» неактивна с пояснением «Класс уже начался — отменить запись нельзя» (UC-2 E1). |
| **Адрес студии: загрузка / ошибка / офлайн** | По `00-foundations.md` §4.5: skeleton при загрузке; при ошибке/офлайн — fallback на текстовый адрес. Остальные детали брони остаются доступны. |

Empty state как таковой неприменим — экран всегда про одну существующую бронь; отсутствие записи трактуется как ошибка/невалидный переход (показ Error).

## 8. Валидации и микрокопия

- **«Отменить» — только до начала** (UC-2 E1). CTA доступен, пока `slot.start_at` в будущем. После начала — недоступен. Пояснение в disabled-состоянии: «Класс уже начался — отменить запись нельзя.»
- **Повторная отмена не выполняется** (UC-2 E2). Для записи со статусом «отменена» / «поздняя отмена» / «отменён студией» CTA скрыт/disabled; показывается текущий статус. Пояснение: «Запись уже отменена.»
- **Правило 2 часов — ссылкой, без дублирования.** Рядом с активной кнопкой «Отменить» — короткая подсказка на правило отмены; полный текст и логика ранней/поздней отмены — в `00-foundations.md` §6 (единый источник) и раскрываются в BS-003. Текст правила здесь не переписывается.
- **Явное время отсечки.** Рядом с правилом показываем конкретный момент отсечки — «Бесплатно освободить место можно до <время начала − 2 ч>» (значение вычисляется из `slot.start_at`, без хардкода). Помогает клиенту сориентироваться, ранняя отмена или поздняя.

Тон — спокойный, без давления и вины (`00-foundations.md` §6, P6).

## 9. Доступность и mobile-first

Базовые требования — `00-foundations.md` §7 и §3. Специфика экрана:

- **Читаемый статус** — бейдж дублирует смысл текстом и иконкой, не только цветом; различим при ярком освещении.
- **Крупный CTA** — кнопка «Отменить» во всю ширину, тач-зона ≥ минимальной; disabled-состояние визуально и программно (доступное имя) считывается screen reader.
- **Осторожность с деструктивным действием** — отмена не происходит по одному тапу; обязательное подтверждение в BS-003. Кнопка не должна провоцировать случайное нажатие «мокрым пальцем».
- Layout не ломается при системном увеличении шрифта; ключевые числа остаются крупными и контрастными.

## 10. Критерии приёмки дизайна

```gherkin
Сценарий: AC-001 Показ всех деталей и статуса
  Дано клиент открыл детали своей записи
  Когда экран в состоянии Content
  Тогда отображаются статус, дата/время начала, программа и её тип,
        шеф, число мест, вариант инвентаря, пищевые ограничения (если указаны),
        цена и дата создания
  И показан адрес студии (текстом) и, опционально, карта
  И для отменённой записи дополнительно отображается дата отмены

Сценарий: AC-007 Адрес студии
  Дано клиент открыл детали своей записи
  Тогда показан текстовый блок с адресом студии
  И (опционально) статичная карта с пином, если API предоставляет координаты
  Когда клиент тапает по адресу или «Открыть карту»
  Тогда открывается шторка BS-004 с полным адресом и, если доступно, интерактивной картой
  И при недоступности карты показан текстовый fallback (адрес + ссылка)

Сценарий: AC-002 CTA «Отменить» доступна для активной предстоящей записи
  Дано запись активна и класс ещё не начался
  Когда экран в состоянии Content
  Тогда кнопка «Отменить» активна (enabled)
  И тап по ней открывает шторку BS-003 «Подтверждение отмены»

Сценарий: AC-003 CTA недоступна для прошедшего класса (UC-2 E1)
  Дано время начала класса уже наступило
  Когда экран в состоянии Content
  Тогда кнопка «Отменить» показана неактивной (disabled)
  И рядом показано пояснение, что отменить запись нельзя

Сценарий: AC-004 CTA недоступна для отменённой записи (UC-2 E2)
  Дано запись уже отменена (ранняя или поздняя отмена)
  Когда экран в состоянии Content
  Тогда кнопка «Отменить» показана неактивной (disabled) с пояснением «Запись уже отменена»
  И отображается текущий статус и дата отмены

Сценарий: AC-005 Запуск подтверждения отмены
  Дано кнопка «Отменить» активна
  Когда клиент нажимает «Отменить»
  Тогда открывается шторка BS-003 с правилом 2 часов
  И отмена не выполняется без подтверждения в шторке

Сценарий: AC-006 Статус считывается без цвета
  Дано экран в состоянии Content
  Когда отображается бейдж статуса
  Тогда смысл статуса передан иконкой и/или текстом, а не только цветом
```

## 11. Решения

| # | Решение |
|---|---------|
| 1 | **Отмена — только целиком.** Частичная отмена отдельных мест в MVP не поддерживается (§6.3). |
| 2 | **Точное время отсечки показываем** — «до <начало − 2 ч>» (вычисляется из `start_at`) (§8). |
| 3 | **Disabled CTA показывается неактивной с обязательным пояснением причины** (не скрывается) (§6.3, §7). |
| 4 | **Пищевые ограничения** отображаются полностью, если были указаны при бронировании. Если не указаны — блок либо скрыт, либо показывает «Не указаны» нейтральным тоном. |
| 5 | **Доп. действия для прошедших записей** (например, «связаться со студией») — вне MVP. |
