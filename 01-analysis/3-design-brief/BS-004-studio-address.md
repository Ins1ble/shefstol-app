
# Требования на дизайн · BS-004 · Адрес студии

**ID:** BS-004 · **Тип:** Bottom Sheet · **Зона:** АЗ · **Приоритет:** Medium · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-06-15

> Шторка показывает адрес кулинарной студии «Шеф-стол» и, при наличии координат, интерактивную карту с отметкой студии. Вызывается с экранов SCR-003 и SCR-006. Сквозные правила (каркас шторок, состояния, доступность, микрокопия) — в `00-foundations.md`; здесь уточняются под этот кейс.

**Источники:** `00-foundations.md` · FR-9a · NFR-26 · US-4

---

## 1. Назначение и контекст

Шторка позволяет клиенту увидеть, где находится студия, и при необходимости построить маршрут до неё. Она раскрывает краткий адресный блок с экранов карточки класса (SCR-003) или деталей брони (SCR-006). Если бэкенд передаёт координаты, шторка показывает интерактивную карту с пином студии; если координат нет — только текстовый адрес и кнопки для внешних карт.

Контекст использования — кухня/студия, мобильный телефон, спешка: карта крупная, управление жестами, важная информация (адрес) дублируется текстом и читается при ярком освещении.

BS-004 — вспомогательный просмотр, не критичное подтверждение: закрытие свободным жестом (свайп, тап по бэкдропу) допустимо.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** понять, как добраться до студии, увидеть точку на карте, запустить навигацию.
- **Цель продукта:** снять неопределённость «куда идти» и предоставить удобный способ построить маршрут (US-4).

---

## 3. Навигация

### Входящие

- Тап по текстовому блоку адреса или мини-карте на SCR-003 «Карточка класса».
- Тап по текстовому блоку адреса или мини-карте на SCR-006 «Детали брони + отмена».

### Исходящие

- **Закрытие** (кнопка «Закрыть», свайп вниз, тап по бэкдропу) → возврат на исходный экран без изменений.
- **«Проложить маршрут»** → открытие внешней навигации (Яндекс.Карты, Apple Maps и т.п.) с построением маршрута до адреса студии.
- **«Открыть в картах»** → открытие адреса студии во внешнем картографическом приложении или веб-версии.

### Свойства шторки

Общие правила bottom sheet — `00-foundations.md` §4.3. Высота шторки — расширенная (до ~90% экрана), чтобы карта была крупной и ей было удобно пользоваться. Поведение закрытия — свободное.

---

## 4. Контент-инвентарь

Все данные приходят из API (read-only). Числа, координаты и адрес не хардкодятся.

| # | Элемент | Источник данных | Назначение |
|---|---------|------------------|------------|
| 1 | Адрес студии (текст) | `studio_address` | Основной текстовый блок с полным адресом. |
| 2 | Координаты студии | `studio_lat`, `studio_lng` (опционально) | Если есть — отображается интерактивная карта с пином. |
| 3 | Пин студии | `studio_lat`, `studio_lng` | Маркер на карте, по тапу — подпись с адресом. |
| 4 | Кнопка «Проложить маршрут» | — | Запуск внешней навигации до координат студии. |
| 5 | Кнопка «Открыть в картах» | — | Открытие координат/адреса во внешнем картографическом приложении. |
| 6 | Кнопка «Закрыть» / грабер | — | Закрытие шторки, возврат на исходный экран. |

Сверху вниз: грабер → хедер (заголовок «Адрес студии» + «Закрыть») → текстовый адрес → опционально интерактивная карта с пином → панель действий.

## 6. Компоненты и поведение

### 6.1 Текстовый адрес

- Полный почтовый адрес студии из поля `studio_address`. Отображается крупно, контрастно, читаем при ярком свете (`00-foundations.md` §3.2).
- Служит текстовым эквивалентом для доступности и fallback при отсутствии карты.

### 6.2 Интерактивная карта (если есть координаты)

- Отображается, если API передал `studio_lat` и `studio_lng`.
- Полотно карты (Яндекс.Карты или аналог, в зависимости от конфигурации) с поддержкой зума и пана.
- На карте — **пин студии** (метка по координатам). По тапу на пин — всплывающая подпись с адресом.
- Контролы зума (＋/−) — крупные тач-зоны (`00-foundations.md` §3.1).
- Ключ API и стиль карты — параметры конфигурации (`00-foundations.md` §4.5).
- Если карта не загрузилась (ошибка, нет сети), вместо неё — нейтральное сообщение и кнопка «Обновить». Текстовый адрес и внешние кнопки остаются доступны.

### 6.3 Кнопки действий

- **«Проложить маршрут»** — открывает внешнее навигационное приложение с построением маршрута от текущего местоположения пользователя до координат студии (или до текстового адреса, если координат нет).
- **«Открыть в картах»** — открывает координаты/адрес в штатном картографическом приложении устройства (handoff).
- Обе кнопки — крупные тач-зоны, имеют доступные имена, состояние нажатия.

### 6.4 Закрытие шторки

- Кнопка «Закрыть» (в хедере), свайп вниз, тап по бэкдропу — все способы возвращают на исходный экран без изменений. BS-004 — не критичное подтверждение, свободный жест закрытия допустим.

## 7. Состояния экрана

Сквозной паттерн — `00-foundations.md` §5 и правила карты §4.5.

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Content (с картой)** | Текстовый адрес, интерактивная карта с пином, кнопки «Проложить маршрут» и «Открыть в картах». |
| **Content (без карты)** | Текстовый адрес, кнопки действий. Карта скрыта. |
| **Loading (инициализация карты)** | На месте карты — скелетон/индикатор загрузки. Текстовый адрес виден. |
| **Error / Offline (карта не загрузилась)** | Вместо карты — нейтральное сообщение «Не удалось загрузить карту» + кнопка «Обновить». Текстовый адрес и внешние кнопки доступны. |

## 8. Валидации и микрокопия

BS-004 не содержит вводимых полей. Контентные тексты:

| Контекст | Текст |
|----------|-------|
| Заголовок | «Адрес студии» |
| Кнопка навигации | «Проложить маршрут» |
| Кнопка внешней карты | «Открыть в картах» |
| Кнопка закрытия | «Закрыть» |
| Ошибка загрузки карты | «Не удалось загрузить карту. Проверьте соединение и попробуйте снова.» (`00-foundations.md` §6) |
| Обновить карту | «Обновить» |

Тон — простой, без давления (`00-foundations.md` §6). Адрес и координаты — из данных, без хардкода.

---

## 9. Доступность и mobile-first

Базовые правила — `00-foundations.md` §7, §3.1, §4.5. Специфика BS-004:

- **Карта не единственный носитель:** адрес студии всегда дублируется текстом. При недоступности карты текстовый адрес и внешние кнопки остаются.
- Крупные контролы (зум, кнопки действий, «Закрыть») с достаточными отступами.
- Контраст текста и элементов карты — не ниже WCAG AA, читаем при ярком свете.
- Все интерактивные элементы имеют доступные имена.
- При увеличении системного шрифта layout не ломается.

## 10. Критерии приёмки дизайна

```gherkin
Функция: Шторка адреса студии (BS-004)

  Сценарий: AC-001 — Открытие с карточки класса
    Дано клиент на SCR-003 и видит адрес студии
    Когда он тапает по адресу
    Тогда снизу открывается BS-004 с текстовым адресом и картой (если есть координаты)
    И на карте отмечен пин студии

  Сценарий: AC-002 — Открытие из деталей брони
    Дано клиент на SCR-006 и видит адрес студии
    Когда он тапает по адресу
    Тогда открывается BS-004 с теми же данными

  Сценарий: AC-003 — Отображение без координат
    Дано API не передал координаты студии
    Когда открывается BS-004
    Тогда карта не показывается, только текстовый адрес
    И кнопки «Проложить маршрут» и «Открыть в картах» доступны

  Сценарий: AC-004 — Зум и пан карты
    Дано карта загружена
    Когда клиент масштабирует и двигает карту
    Тогда карта реагирует на жесты, пин остаётся видимым

  Сценарий: AC-005 — Переход к навигации
    Дано открыта BS-004
    Когда клиент нажимает «Проложить маршрут»
    Тогда открывается внешнее приложение с построением маршрута до студии

  Сценарий: AC-006 — Открыть в картах
    Дано открыта BS-004
    Когда клиент нажимает «Открыть в картах»
    Тогда координаты/адрес открываются во внешнем картографическом приложении

  Сценарий: AC-007 — Закрытие
    Дано BS-004 открыта
    Когда клиент свайпает вниз, тапает по бэкдропу или жмёт «Закрыть»
    Тогда шторка закрывается, возврат на исходный экран

  Сценарий: AC-008 — Ошибка загрузки карты
    Дано карта не загрузилась (сеть, ошибка API)
    Тогда вместо карты показано сообщение «Не удалось загрузить карту» и кнопка «Обновить»
    И текстовый адрес и кнопки действий доступны
```

## 11. Решения

| # | Решение |
|---|---------|
| 1 | BS-004 — отдельная шторка, открывается по тапу на адрес с SCR-003/SCR-006. |
| 2 | Координаты студии (`studio_lat`, `studio_lng`) опциональны. Если они есть — показывается интерактивная карта с пином. Если нет — только текстовый адрес и внешние кнопки. |
| 3 | Навигация и просмотр в картах — handoff во внешние приложения. Turn-by-turn внутри приложения — вне скоупа MVP. |
| 4 | Высота шторки — расширенная (до ~90%) при наличии карты, стандартная по контенту — если только текст. |
| 5 | Ключ API карт — параметр конфигурации, в макет не зашивается. |
