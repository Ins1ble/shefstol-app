## Обзор

**SCR-002 «Список классов» — главный экран приложения «Шеф-стол» и корневой таб авторизованной зоны.** Открывается сразу после успешного входа и доступен по табу «Классы». Показывает каталог предстоящих кулинарных классов и служит единственной точкой входа в сценарий записи: отсюда клиент открывает карточку класса и уточняет выдачу через фильтры.

Список загружается запросом `listSlots` (`GET /slots`) — read-only-проекция расписания. По умолчанию (`only_available=false`, период дат не задан) показываются классы на **ближайшие 7 дней** (дефолт API: `date_from = now`, `date_to = now + 7 дней`), отсортированные по `start_at` по возрастанию (ближайшие первыми) и сгруппированные по дням со sticky-заголовками. Больший период — через явный фильтр дат.

Каждая карточка класса содержит:
- Дату/время начала
- Название программы (меню) и её тип (новичковая / опытная)
- Имя шефа
- Цену за место
- Доступность мест («Свободно N из M»)
- При отсутствии мест — бейдж «Мест нет», карточка некликабельна

В хедере — иконка «Фильтры» с индикатором активных фильтров (открывает шторку BS-001). Жест pull-to-refresh обновляет список и актуальную доступность мест. Экран подчиняется сквозному паттерну состояний (LOGIC-008) и работает в режиме read-only.

### User Story

> Как клиент, я хочу видеть список предстоящих кулинарных классов, чтобы выбрать подходящий и записаться. (US-2)
>
> Как клиент, я хочу фильтровать список классов, чтобы быстро находить подходящий вариант. (US-3)

### Бизнес-ценность

- Короткий путь к записи: главный экран сразу показывает каталог классов и ведёт в запись за минимум шагов (NFR-2).
- Прозрачность доступности: заполненные классы видны с пометкой «Мест нет», что снижает путаницу и заменяет ручную запись через WhatsApp/Google-таблицу.
- Предсказуемость выдачи: единый порядок (ближайшие первыми) и группировка по дням упрощают ориентацию в расписании.
- Воспринимаемая скорость: скелетоны и pull-to-refresh дают ощущение мгновенного отклика (NFR-6, P4).

## Используемые запросы

### listSlots

**Тип:** REST  
**Метод:** GET  
**Спецификация:** `../api/slots/api.yaml` → `listSlots` (`GET /slots`)

**Триггер:** Инициализация экрана, pull-to-refresh, возврат из BS-001 после «Применить».

**Заголовки:**

| Поле | Описание |
|------|----------|
| `Authorization` | Bearer-токен пользователя |
| `deviceuuid` | Идентификатор устройства |

**Параметры (query):**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `date_from` | string (date-time) | Нет | `appliedFilters.date_from` | Начало периода старта; опускается, если не задано. При отсутствии обоих параметров применяется **дефолт API «ближайшие 7 дней»** (`date_from = now`). |
| `date_to` | string (date-time) | Нет | `appliedFilters.date_to` | Конец периода старта; опускается, если не задано (дефолт API — `now + 7 дней`). Больший период — явным фильтром дат. |
| `program_type` | array(enum) `beginner`/`experienced` | Нет | `appliedFilters.program_type` | Типы программ, OR внутри группы; опускается, если массив пуст. |
| `chef_id` | array(uuid) | Нет | `appliedFilters.chef_id` | Шефы, OR внутри группы; опускается, если массив пуст. |
| `only_available` | bool | Нет | `appliedFilters.only_available` | Только классы со свободными местами; по умолчанию `false`. |
| `limit` | int | Нет | Конфигурация | Размер страницы (1..100, по умолчанию из API). В MVP список грузится целиком. |
| `offset` | int | Нет | Состояние | Смещение страницы; `0` для первой страницы, увеличивается при догрузке (задел на будущее). |

> **Вне MVP:** пагинация / бесконечный скролл (догрузка следующих страниц по `limit`/`offset` с лоадером внизу списка) в MVP **не реализуется** — список грузится целиком одной страницей; параметры `limit`/`offset` оставлены как задел на будущее.

**Ответ:** `SlotListResponse` — `items: SlotSummary[]` + `meta: PaginationMeta` (`limit`, `offset`, `total`).
Поля `SlotSummary`: `id`, `start_at`, `program` (`name`, `type`, `capacity_cap`), `chef` (`name`), `price`, `rental_price`, `total_seats`, `free_seats`, `free_rental_kits`, `status`.

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Скелетоны карточек классов (форма будущих карточек), не пустой экран |
| Загрузка (pull-to-refresh) | `isRefreshing = true` | Refreshing — Loading поверх контента (LOGIC-008, Шаг 1): индикатор обновления сверху списка, список не сбрасывается в скелетон |
| Успех | `items` не пуст | Сортировка по `start_at` ↑, группировка по дням (sticky-заголовки); заполненные классы с бейджем «Мест нет» и некликабельны; обновление индикатора активных фильтров |
| Успех | `items` пуст, фильтры в дефолте | Empty «нет классов вообще»: заголовок «Пока нет доступных классов» + подсказка «Потяните вниз, чтобы обновить» (каталог Empty в LOGIC-008 Шаг 3) |
| Успех | `items` пуст, применены фильтры | Empty «нет по фильтрам»: заголовок «Ничего не найдено по фильтрам» + CTA «Изменить фильтры» (открывает BS-001); тексты — каталог Empty в LOGIC-008 Шаг 3 (UC-3 E1) |
| HTTP 400 | Некорректные параметры (напр. `date_from` > `date_to`) | Снек с текстом из `message`; если `message` пуст — дефолт «Не удалось выполнить. Попробуйте ещё раз.» (`00-foundations` §6); некорректный диапазон дат блокируется на стороне UI до запроса (LOGIC-005) |
| HTTP 401 | Токен отсутствует/неверен | Сценарий повторной авторизации |
| HTTP 5xx | — | Error state с кнопкой «Обновить» |
| Сеть (первичная загрузка) | Нет соединения | Error state с кнопкой «Обновить» (`00-foundations` §6) |
| Ошибка при pull-to-refresh | Сбой сети/сервера при `isRefreshing = true` | Контент сохраняется, экран **не** переходит в Error; ненавязчивый снек «Не удалось обновить. Проверьте соединение и попробуйте снова.» (каталог снеков LOGIC-008 Шаг 6) |

> **Снек успеха применения/сброса фильтра не показывается** (`00-foundations` §6.1, LOGIC-005): обратная связь — обновлённый список и индикатор активных фильтров, дублировать снеком нельзя. Успешный pull-to-refresh снеком тоже не подтверждается — только ошибка обновления (строка выше).

### 1. Хедер

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Заголовок «Классы» | Название корневого таба | — (статичный) | — | — |
| Иконка «Фильтры» | Кнопка открытия шторки фильтров | — | — | Открыть BS-001, передать `appliedFilters` |
| Индикатор активных фильтров | Бейдж-счётчик у иконки «Фильтры»: компактная форма (кружок/пилюля) с числом активных групп фильтров (≠ дефолт). Носитель смысла — форма + число, не только цвет (NFR-1). | `appliedFilters` | — | — |

**Логика:**
- Иконка «Фильтры» / индикатор: LOGIC-005 — индикатор виден, если хотя бы один применённый фильтр отличается от дефолта; скрыт, когда все фильтры в дефолте. **Условия доступности:**
- Индикатор активных фильтров видим, если: хотя бы один из `appliedFilters` ≠ дефолт.

**Логика:**
- Список: LOGIC-005 — сортировка по `start_at` ↑, группировка по дням со sticky-заголовками; классы с `free_seats = 0` — бейдж «Мест нет», некликабельны.
- Состояния списка: LOGIC-008 — Loading/Content/Empty/Error, pull-to-refresh.

**Условия доступности:**
- Карточка класса кликабельна (открывает SCR-003), если: `free_seats > 0`.
- Карточка класса некликабельна (бейдж «Мест нет»), если: `free_seats = 0`.

### 3. Пустые состояния (Empty)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Заголовок Empty «нет классов вообще» | «Пока нет доступных классов» | — | — | — |
| Подсказка Empty «нет классов вообще» | «Потяните вниз, чтобы обновить» | — | — | — (обновление — жестом pull-to-refresh) |
| Заголовок Empty «нет по фильтрам» | «Ничего не найдено по фильтрам» | — | — | — |
| CTA «Изменить фильтры» | Кнопка в Empty «нет по фильтрам» | — | — | Открыть BS-001, передать `appliedFilters` |

**Логика:**
- Empty: LOGIC-008 Шаг 3 — две разновидности Empty (тексты и CTA — по каталогу Empty); LOGIC-005 — пустой результат по фильтрам (UC-3 E1).

**Условия доступности:**
- Empty «нет классов вообще» — при пустом `items` и фильтрах в дефолте (CTA нет, только подсказка про обновление).
- Empty «нет по фильтрам» с CTA «Изменить фильтры» — при пустом `items` и хотя бы одном применённом фильтре.

### 4. Обновление и таб-бар

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Pull-to-refresh | Жест «потянуть вниз» | — | — | Повторный запрос [listSlots](#listslots), обновление доступности |
| Таб «Классы» | Текущий таб | — | — | — (активный) |
| Таб «Мои записи» | Переход к броням | — | — | Открыть SCR-005 |
| Таб «Профиль» | Переход в профиль | — | — | Открыть SCR-007 |

**Логика:**
- Pull-to-refresh: LOGIC-008 — Loading поверх контента (индикатор обновления), список не сбрасывается в скелетон; счётчики `free_seats` обновляются актуальными данными.

## Состояния экрана

### Таблица состояний

| Состояние | Условие | Отображение |
|-----------|---------|-------------|
| Loading | Ожидание `listSlots` (первичная загрузка) | Скелетоны карточек классов в форме будущего контента |
| Refreshing | Pull-to-refresh, `isRefreshing = true` | Не самостоятельное состояние, а Loading поверх контента по LOGIC-008 (Шаг 1, Refreshing): индикатор обновления сверху списка, контент **не сбрасывается** в скелетон. Ошибка обновления → снек, экран не уходит в Error |
| Content | 200 + `items` не пуст | Список карточек (со свободными и заполненными — последние с «Мест нет»), сортировка по `start_at`, секции по дням |
| Empty (нет классов вообще) | 200 + `items` пуст, фильтры в дефолте | Заглушка: «Пока нет доступных классов» + «Потяните вниз, чтобы обновить» (обновление — жестом pull-to-refresh) |
| Empty (нет по фильтрам) | 200 + `items` пуст, применены фильтры | Заглушка: «Ничего не найдено по фильтрам» + CTA «Изменить фильтры» (открывает BS-001); каталог Empty LOGIC-008 Шаг 3 (UC-3 E1) |
| Error | 4xx (кроме 400-валидации) / 5xx / таймаут / нет сети при первичной загрузке | Заглушка ошибки + кнопка «Обновить» (`00-foundations` §6). Ошибка при pull-to-refresh в Error не переводит — см. обработку `listSlots` |

### Диаграмма переходов

```mermaid
stateDiagram-v2
    [*] --> Loading : onEnter / таб «Классы»

    Loading --> Content : 200 + items не пуст
    Loading --> EmptyNoSlots : 200 + items пуст, фильтры дефолт
    Loading --> EmptyByFilters : 200 + items пуст, фильтры применены
    Loading --> Error : 4xx / 5xx / таймаут / нет сети

    Content --> Loading : pull-to-refresh / возврат из BS-001 «Применить»
    Content --> Content : ошибка pull-to-refresh (контент сохранён) + снек
    EmptyNoSlots --> Loading : pull-to-refresh
    EmptyByFilters --> Loading : CTA «Изменить фильтры» → BS-001 «Применить»
    Error --> Loading : тап «Обновить»

    Content --> [*] : тап по классу / таб / навигация
```

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** клиент авторизован и открыл таб «Классы», фильтры в дефолте, **Когда** `listSlots` вернул непустой список, **Тогда** показаны классы на **ближайшие 7 дней** (дефолт API), каждая карточка содержит дату/время начала, программу и тип, шефа, цену и «Свободно N из M». | P0 |
| AC-002 | **Дано** на экране есть карточка класса с `free_seats > 0`, **Когда** клиент тапает в любую область карточки, **Тогда** открывается SCR-003 для этого класса с `slotId`. | P0 |
| AC-003 | **Дано** список содержит классы на разные даты, **Когда** список загружен, **Тогда** классы отсортированы по `start_at` по возрастанию (ближайшие первыми) и сгруппированы по дням со sticky-заголовками. | P0 |
| AC-004 | **Дано** клиент на экране списка, **Когда** клиент тапает иконку «Фильтры», **Тогда** открывается шторка BS-001; **И** после применения хотя бы одного фильтра в хедере виден бейдж-счётчик активных фильтров (форма + число, не только цвет); снек об успешном применении фильтра не показывается (`00-foundations` §6.1). | P1 |
| AC-005 | **Дано** список отображается, **Когда** клиент тянет список вниз (pull-to-refresh), **Тогда** список и счётчики `free_seats` обновляются актуальными данными, во время обновления виден индикатор, контент не сбрасывается в скелетон. | P1 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** запрос `listSlots` завершился ошибкой 5xx или нет сети, **Когда** отображается экран, **Тогда** показана заглушка ошибки с кнопкой «Обновить»; **И** тап «Обновить» возвращает экран в Loading и повторяет запрос. | P0 |
| AC-N02 | **Дано** у класса `free_seats = 0` и фильтр «только со свободными местами» выключен, **Когда** список загружен, **Тогда** класс показан с бейджем «Мест нет» (текст+форма) и некликабелен — тап не открывает SCR-003. | P0 |
| AC-N03 | **Дано** применены фильтры, под которые нет ни одного класса, **Когда** `listSlots` вернул пустой `items`, **Тогда** показан empty state «Ничего не найдено по фильтрам» с CTA «Изменить фильтры», открывающим BS-001 (UC-3 E1, каталог Empty LOGIC-008 Шаг 3). | P1 |
| AC-N04 | **Дано** передан некорректный диапазон дат (`date_from` > `date_to`), **Когда** UI пытается сформировать запрос, **Тогда** некорректный диапазон блокируется до запроса; при ответе 400 от сервера показывается снек с текстом из `message`, а если `message` пуст — дефолт «Не удалось выполнить. Попробуйте ещё раз.» (`00-foundations` §6). | P2 |
| AC-N05 | **Дано** список отображается, **Когда** pull-to-refresh завершился ошибкой (сеть/сервер), **Тогда** контент сохраняется, экран не переходит в Error, показан ненавязчивый снек «Не удалось обновить. Проверьте соединение и попробуйте снова.». | P1 |

### Граничные условия (Edge Cases)

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** в системе нет ни одного предстоящего класса и фильтры в дефолте, **Когда** `listSlots` вернул пустой `items`, **Тогда** показан empty state «Пока нет доступных классов» с подсказкой «Потяните вниз, чтобы обновить» (без CTA «Изменить фильтры»; обновление — жестом pull-to-refresh). | P1 |
| AC-E02 | **Дано** экран открывается и данные ещё не получены, **Когда** идёт первичная загрузка, **Тогда** показаны скелетоны карточек, а не пустой экран; отклик ощущается в пределах 2–3 секунд (NFR-6). | P1 |
| AC-E03 | **Дано** класс заполнился, пока клиент смотрел список (его `free_seats` стал 0), **Когда** клиент делает pull-to-refresh, **Тогда** карточка обновляется бейджем «Мест нет» и становится некликабельной без перехода в карточку. | P2 |
| AC-E04 | **Дано** включён фильтр «только со свободными местами» (`only_available=true`), **Когда** список загружен, **Тогда** классы с `free_seats = 0` отсутствуют в выдаче (бейдж «Мест нет» не показывается). | P2 |
| AC-E05 | **Дано** включено системное увеличение шрифта, **Когда** отображается карточка класса, **Тогда** layout карточки не ломается, ключевые числа (время, цена, свободно мест) остаются читаемы и контрастны (NFR-1). | P2 |
