
# 09. Логики — индекс

Переиспользуемая бизнес- и UI-логика приложения «Шеф-стол». Выносится один раз и подключается на экранах через секцию «Применяемые логики» по ссылке (принцип DRY). Шаблон — `_LOGIC_TEMPLATE.md`.

**Статус:** Черновик · **Дата:** 2026-07-05

## Реестр логик

| ID | Логика | Приоритет | Назначение | Применяется на |
|----|--------|-----------|------------|----------------|
| LOGIC-001 | OTP-авторизация и сессия | Critical | Вход по телефону без пароля (3 шага), управление сессией/токеном, выход | SCR-001, SCR-007 |
| LOGIC-002 | Расчёт доступности | Critical | `max_seats = min(free_seats, program.capacity_cap, 3)` + проверка прокатного фонда `rental_count ≤ free_rental_kits` | SCR-003, SCR-004 |
| LOGIC-003 | Расчёт цены брони | High | `price × seats + rental_price × rental`, разбивка, офлайн-оплата | SCR-003, SCR-004, BS-002, SCR-005, SCR-006 |
| LOGIC-004 | Отмена: правило 2 часов | Critical | Ранняя (≥2 ч) / поздняя (<2 ч) отмена, возврат мест и прокатных наборов | SCR-006, BS-003 |
| LOGIC-005 | Фильтрация и сортировка классов | High | Фильтры (дата/тип программы/шеф/доступность), OR внутри / AND между группами | SCR-002, BS-001 |
| LOGIC-006 | Адрес студии | Medium | Текстовый адрес + опциональная карта с fallback, handoff во внешние карты | SCR-003, SCR-006, BS-004 |
| LOGIC-007 | Запрос push-разрешения | Medium | Системный запрос push только после первой брони, регистрация токена | BS-002 |
| LOGIC-008 | Паттерн состояний экрана | High | Loading → Content → Empty → Error + pull-to-refresh + каталог снеков | Все экраны с запросами |
