# Бизнес-требования

Документ фиксирует бизнес-цели, метрики успеха и ограничения верхнего уровня для клиентского мобильного приложения кулинарной студии «Шеф-стол». Основан на брифе заказчика, уточняющих вопросах и описании домена. Служит фундаментом для функциональных требований, пользовательских историй и приоритизации MVP.

**Скоуп.** В поставку входит только клиентское мобильное приложение и API для него. Единственная роль в приложении — **Клиент**. Интерфейсы шефа, администратора/владельца, управление расписанием и операционные процессы реализованы в существующей инфраструктуре студии и в текущий скоуп не включены (R-028).

**Нумерация.** Идентификаторы требований (BR-NN, M-NN) зафиксированы для сквозной трассировки на всех этапах. Возможные пропуски в нумерации (например, BR-3, BR-4, BR-6) — следствие осознанного сужения скоупа и переноса отдельных инициатив в Phase 2.

## Метрики успеха

| ID | Метрика | Целевой ориентир | Связь с требованиями |
|----|---------|-------------------|----------------------|
| **M-1** | Доля записей, сделанных клиентами онлайн самостоятельно (от общего числа записей за сезон) | ≥ 70% к концу первого сезона | BR-2 |
| **M-2** | Число двойных броней / конфликтов по местам | 0 (полное устранение) | BR-1 |
| **M-4** | Простой столов из-за поздних отмен (доля поздних отмен от общего числа записанных) | Снижение относительно текущего уровня; целевой ориентир ≤ 10% | BR-5 |
| **M-6** | Готовность к старту сезона | Запуск в срок ~2 месяцев с критичным набором функций (просмотр слотов, запись, отмены) | BR-7 |

## Ограничения верхнего уровня

**Срок и бюджет.** Жёсткая цель — готовность MVP к началу сезона (~2 месяца, ориентир 1 сентября). Бюджет ограничен — применяется строгий MVP-подход. Приоритет первого запуска: безошибочная запись, прокат инвентаря, push-напоминания (customer-questions, №35).

**Оплата.** Исключительно офлайн на старте (наличные / перевод на карту на месте). Приложение отображает итоговую стоимость брони и фиксирует запись без предоплаты. Онлайн-оплата и платёжные интеграции — вне границ MVP, закладываются в развитие (BR-8).

**Платформа.** Mobile-first приложение для iOS и Android, русский интерфейс. Данные (слоты, программы, шефы) потребляются через API в режиме read-only. Бэкенд выступает как black-box источник истины и гарантирует атомарность бронирования (BR-9, R-004, R-015).

**Вне скоупа.** Функции шефа и владельца/администратора, управление расписанием, назначение персонала, отмена класса целиком (выполняется в существующей админке), онлайн-оплата, программа лояльности, публичные отзывы, SMS-уведомления.

**Оценки шефов — Phase 2** (подтверждено заказчиком). Функция оценок осознанно вынесена за пределы MVP: требует интерфейса владельца и не укладывается в жёсткий срок первой поставки. Заказчик согласовал перенос; в backlog MVP функция не включается.
