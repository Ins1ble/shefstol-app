# Описание домена и ограничения

Документ фиксирует предметную область кулинарной студии «Шеф-стол» и границы клиентского мобильного приложения. Источники: бриф заказчика и ответы на уточняющие вопросы. Используется как база для формулирования требований, use cases и контракта API.

## Описание домена

**Бизнес.** Кулинарная студия «Шеф-стол» работает третий год в лофте на бывшем заводе. Проводит групповые кулинарные классы длительностью ~3 часа. Две основные линейки программ: простая итальянская кухня для новичков и более сложные классы для опытных. Классы ведут шеф-повара — постоянный штат из 5 человек, в сезон подключаются приглашённые. В студии 12 рабочих столов; клиентам доступен прокат комплектов «фартук + ножи» (15 наборов) или использование своего оборудования.

**Процесс.** Сейчас запись ведётся вручную через WhatsApp и Google-таблицу: владелец проверяет свободные места, шефов и отвечает клиентам. В пиковые дни возникают двойные брони, путаница с столами и закупкой продуктов. С клиентским приложением клиент самостоятельно просматривает расписание и бронирует места; владелец контролирует загрузку через существующую инфраструктуру.

**Цель клиентского приложения.** Дать клиенту удобную онлайн-запись на классы, исключить двойные брони и путаницу с местами. Формирование расписания, назначение шефов, отмена класса целиком и операционное управление остаются в существующей инфраструктуре студии (бэкенд, админка, интерфейсы шефа).

**Ключевые объекты домена:**

- **Класс / слот** — конкретное проведение программы в заданное время с назначенным шефом, лимитом мест и числом свободных мест. Единица записи клиента.
- **Программа (меню)** — тема и содержание класса (например, итальянская паста для новичков), уровень сложности, потолок вместимости (12 или 8 мест для классов с духовкой/сложной техникой).
- **Шеф** — повар, ведущий класс. Отображается клиенту в карточке слота; назначение — на стороне бэкенда.
- **Рабочее место (стол)** — одно место за столом на одного человека. Лимит слота определяется числом свободных столов и потолком программы.
- **Прокатный набор (фартук + ножи)** — комплект студии, выдаваемый напрокат. Фонд — 15 наборов на класс; учитывается отдельно от рабочих мест.
- **Запись (бронь)** — резервирование одного или нескольких мест (до 3) на слот с выбором проката или своего оборудования, указанием пищевых ограничений и итоговой ценой.
- **Клиент** — пользователь приложения, записывающийся на классы, управляющий своими бронями и получающий push-уведомления.

## Пользователи и роли

В рамках текущей поставки разрабатывается **только клиентское мобильное приложение и API для него**. Роли «Шеф» и «Владелец / администратор» реализованы в существующей инфраструктуре и в скоуп не входят.

| Роль | Описание | Цели | В скоупе |
|------|----------|------|----------|
| **Клиент** | Посетитель студии, записывающийся на кулинарные классы через мобильное приложение | Просматривать и фильтровать слоты; записаться на класс (себя и до 2 гостей); выбрать прокат или своё оборудование для каждого места; указать пищевые ограничения; видеть предстоящие брони и историю; отменять запись по правилам; получать push-напоминания и уведомления об изменениях | Да |

## Ограничения

### Бизнес-ограничения

- В студии **12 рабочих столов**; для классов с духовкой и сложной техникой — **не более 8 человек** (ограничены плитами).
- Лимит мест слота = **min(свободные столы, потолок программы)**. Один стол = одно место = один человек.
- **Прокатных наборов — 15**; выбор «своё оборудование» прокатный фонд не расходует.
- **Бесплатная отмена** клиентом за **≥2 часа** до начала — место освобождается. **Поздняя отмена** (<2 ч) фиксируется со статусом «Поздняя отмена», место не освобождается, штрафов нет.
- **Оплата офлайн** (наличные / перевод на карту на месте). Приложение показывает итоговую цену (место + прокат) и фиксирует бронь без предоплаты. Онлайн-оплата — вне MVP.
- Период показа слотов по умолчанию — **7 дней** (R-027); более длинный период — через фильтр дат. При отсутствии слотов — empty state **«Пока нет доступных классов»**.
- **Отмена класса студией** (форс-мажор: срыв поставки, поломка оборудования): бронь не удаляется, получает статус **«Отменён студией»** с причиной; клиенту — push; повторная запись на этот слот запрещена (R-008). Сама отмена слота выполняется в существующей инфраструктуре.

### Технические / платформенные

- **Mobile-first**, платформы **iOS + Android**; интерфейс на русском языке.
- Клиентское приложение взаимодействует с существующим бэкендом **через API**; слоты, программы, шефы — **read-only** для клиента.
- Бэкенд — **black-box** и источник истины; атомарность бронирования и гарантия **0 двойных броней** — на его стороне. Клиент полагается на контракт API и корректно обрабатывает отказ при отсутствии мест.
- Проект учебный/тестовый, **легаси-данных нет**; каноническая схема данных — **контракт API** (R-015).
- Авторизация клиента — **OTP по номеру телефона**. Push-канал — **только APNs/FCM**, без SMS.

### Проект

- Срок поставки MVP — **к началу сезона** (~2 месяца; ориентир — 1 сентября).
- **Бюджет ограничен**; приоритет MVP: (1) запись без двойных броней, (2) прокат оборудования, (3) push-напоминания.

## Границы скоупа

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
|--------------------------------|-------------------------------------------------------------|
| Регистрация и вход по телефону (OTP) | Интерфейсы шефа и администратора / владельца |
| Просмотр и фильтрация слотов (дефолт 7 дней) | Формирование и редактирование расписания |
| Запись на класс (до 3 мест в одной брони) | Назначение шефов на слоты |
| Выбор прокат / своё оборудование для каждого места | Отмена класса целиком (форс-мажор) — действие в админке |
| Необязательное поле «Пищевые ограничения» при бронировании | Онлайн-оплата и платёжные интеграции |
| Список своих броней (предстоящие + история) | Оценки шефов (Phase 2) |
| Отмена брони клиентом (правило 2 часов) | Программа лояльности (Phase 2) |
| Отображение итоговой цены; пометка об офлайн-оплате | Публичные отзывы |
| Push-уведомления: подтверждение, напоминания за 24 ч и 2 ч, изменения, отмена студией | SMS-уведомления |
| Добавление класса в календарь телефона | Поделиться записью ссылкой |
| Обработка статуса «Отменён студией» и push клиенту | Возврат / компенсация при отмене студией (решается offline) |

## Глоссарий

| Термин | Определение |
|--------|-------------|
| **Класс / слот** | Запланированное проведение программы в конкретное время с шефом, лимитом и свободными местами |
| **Программа (меню)** | Тема класса, уровень сложности и потолок вместимости (до 12 или до 8 мест) |
| **Шеф** | Повар, ведущий класс; назначается в существующей инфраструктуре |
| **Рабочее место (стол)** | Одно место за столом на одного участника класса |
| **Прокатный набор** | Комлект студии «фартук + ножи»; фонд 15 наборов, учитывается отдельно от мест |
| **Своё оборудование** | Вариант бронирования, при котором клиент использует свой фартук и ножи; прокатный фонд не расходуется |
| **Запись (бронь)** | Резервирование 1–3 мест на слот с выбором инвентаря, пищевых ограничений и фиксацией цены |
| **Отмена клиентом (ранняя)** | Отмена за ≥2 ч до начала; место освобождается |
| **Отмена клиентом (поздняя)** | Отмена менее чем за 2 ч до начала; статус «Поздняя отмена», место не освобождается, штрафов нет |
| **Отмена студией** | Отмена класса организатором (форс-мажор); бронь получает статус «Отменён студией» с причиной; повторная запись на слот запрещена |
| **Администратор / владелец** | Артём и операционный персонал; управляют расписанием и классами через существующую админку |
| **MVP** | Минимально жизнеспособная версия клиентского приложения к старту сезона; без онлайн-оплаты, оценок и лояльности |
| **Пищевые ограничения** | Необязательное текстовое поле при бронировании (аллергии, диеты, пожелания); передаётся шефу через бэкенд |
