# Фича: Регистрация / Вход (SCR‑001)

## Цель
Предоставить клиенту возможность быстро войти или зарегистрироваться по номеру телефона без пароля, используя одноразовый SMS‑код.

## Требования
- FR‑1, FR‑2, FR‑43
- LOGIC‑001 (OTP‑авторизация и сессия)
- US‑1

## Реализация
- Экран `screen-login` (рендерится функцией `renderLogin()` в `auth.js`).
- Три шага: телефон → код → имя (только для новых).
- Валидация E.164, таймер повторной отправки 30 с.
- Вызовы API: `POST /auth/request-code`, `POST /auth/verify-code`, `PATCH /profile`.
- Сохранение токенов через `setTokens()` для работы автопродления сессии.
- При успехе – переход на главный экран с таб‑баром.

## Отправленные промпты (пример)
- «Реализуй экран входа с полем телефона, кнопкой “Получить код”, полем OTP и таймером повторной отправки»
- «Добавь вызов API /auth/request-code и /auth/verify-code, обработку ошибок»
- «При успешном входе сохраняй токены в localStorage и вызывай setTokens»

## Ручная проверка
1. Открыть приложение (неавторизованным) – виден экран входа.
2. Ввести валидный номер (например, +79991234567) – кнопка активна.
3. Нажать «Получить код» – появляется поле OTP, таймер 0:30.
4. Взять код из консоли сервера, ввести – для нового номера открывается поле «Имя», для существующего – сразу главный экран.
5. После ввода имени и сохранения – переход на список классов.
