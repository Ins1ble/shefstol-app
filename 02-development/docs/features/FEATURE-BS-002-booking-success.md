# Фича: Подтверждение записи (BS‑002)

## 1. Цель
После успешного создания брони показать клиенту однозначное подтверждение со сводкой брони, напоминанием об офлайн-оплате и кнопками для дальнейших действий («Мои записи», «Добавить в календарь», «Готово»). При первой успешной записи инициировать системный запрос разрешения на push-уведомления (LOGIC‑007).

## 2. Требования (источники)
- **Функциональные:** FR‑30 (показ цены, офлайн‑оплата), FR‑33 (напоминания, push)
- **Логики:** LOGIC‑003 (цена), LOGIC‑007 (push-разрешение)
- **User Stories:** US‑5 (подтверждение записи), US‑12 (получать напоминания)
- **Экранные требования:** BS‑002 (`BS-002-booking-success.md`)

## 3. Реализация
### 3.1. Файлы
- `frontend/js/booking.js` – функции `showBookingSuccess`, `addToCalendar`, `goBackToSlotsFromSuccess`, `goToBookings`
- `frontend/js/api.js` – вызов `GET /api/bookings/{id}` для получения деталей сводки

### 3.2. Поток
1. **Открытие экрана успеха**
   - После получения `201 Created` от `POST /bookings` вызывается `showBookingSuccess(booking)`.
   - Отображается полноэкранный блок с иконкой ✓ и заголовком «Вы записаны».
   - Сразу после отрисовки отправляется `GET /api/bookings/{booking.id}` для получения деталей (дата, программа, шеф, места, инвентарь, цена).

2. **Сводка брони**
   - После получения `bookingDetail`:
     - Дата/время начала (`formatDateTime`).
     - Название программы, имя шефа.
     - Количество мест и число прокатных наборов.
     - Пищевые ограничения (если указаны).
     - Итоговая стоимость (`price_total`, крупно, контрастно).
     - Напоминание «Оплата на месте: наличные или перевод».

3. **Кнопки действий**
   - **«Мои записи»** (primary) – переход на вкладку «Мои записи» (SCR‑005).
   - **«Добавить в календарь»** – открывает Google Календарь с предзаполненными датой, временем и названием класса. Кнопка скрыта до получения деталей брони, чтобы передать корректные `start_at` и `program_name`.
   - **«Готово»** (secondary) – возврат на список классов (SCR‑002).

4. **Запрос push-уведомлений (LOGIC‑007)**
   - Проверяются условия: `is_first_booking === true` и `push_permission_requested` отсутствует в `localStorage`.
   - Если условия выполнены, после отрисовки сводки появляется подводка «Напомним за N часов до начала» (часы из `reminder_hours`).
   - Вызывается `Notification.requestPermission()`. При согласии разрешение предоставлено, при отказе ничего не блокируется.
   - Флаг `push_permission_requested` сохраняется в `localStorage`, чтобы запрос не повторялся.
