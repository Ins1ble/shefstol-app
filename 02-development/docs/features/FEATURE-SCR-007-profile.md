# Фича: Профиль клиента (SCR‑007)

## 1. Цель
Предоставить клиенту доступ к его контактным данным (имя, телефон) с возможностью быстрого инлайн-редактирования. Обеспечить безопасный выход из аккаунта и необратимое удаление учётной записи с освобождением забронированных мест. Вся функциональность строго ограничена данными текущего пользователя.

## 2. Требования (источники)
- **Функциональные:** FR‑47 (просмотр и редактирование имени и телефона), FR‑48 (удаление аккаунта с аннулированием броней и анонимизацией), FR‑49 (выход из аккаунта)
- **Логики:** LOGIC‑001 (OTP‑авторизация – смена телефона подтверждается кодом из SMS), LOGIC‑008 (состояния экрана)
- **User Stories:** US‑1 (быстрый вход без барьеров – профиль как точка управления сессией)
- **Экранные требования:** SCR‑007 (`SCR-007-profile.md`)
- **Нефункциональные:** NFR‑11 (защита ПДн), NFR‑12 (доступ только к своим данным)

## 3. Реализация
### 3.1. Файлы
- `frontend/js/profile.js` – функции `loadProfile`, `renderProfile`, `editName`, `saveName`, `editPhone`, `requestPhoneChangeCode`, `confirmPhoneChange`, `confirmLogout`, `executeLogout`, `confirmDeleteAccount`, `executeDeleteAccount`
- `frontend/js/api.js` – вызовы `GET /profile`, `PATCH /profile`, `POST /profile/phone/request-code`, `POST /profile/phone/confirm`, `POST /auth/logout`, `DELETE /profile`
- `frontend/js/utils.js` – скелетон `profile`, снек‑бар

### 3.2. Загрузка и отображение
- При переключении на вкладку «Профиль» вызывается `loadProfile()` → `GET /api/profile`.
- В состоянии **Loading** отображается скелетон профиля (две строки «лейбл/значение»).
- При успехе рендерится:
  - **Имя** и **Телефон** (маскированный) с иконками карандаша `✎` для инлайн-правки.
  - **Справочные пункты:** «Правила студии», «Поддержка» (ссылки, показывают информационный снек), «Версия приложения 1.0.0».
  - Кнопка **«Выйти»** (безопасная зона, отделена отступом).
  - Кнопка **«Удалить аккаунт»** (деструктивная, менее акцентная).

### 3.3. Редактирование имени
- Тап по карандашу `✎` рядом с именем переводит поле в режим редактирования (инлайн).
- Появляется текстовое поле и иконка подтверждения `✓`.
- Валидация: 1–100 символов (локальная + серверная).
- При сохранении → `PATCH /profile { name }`. При успехе – снек «Профиль обновлён» и перерисовка данных.
- При ошибке – снек с текстом ошибки или «Проверьте имя — кажется, тут лишние символы».

### 3.4. Смена телефона
- Аналогично переводится в режим редактирования.
- Валидация E.164. При подтверждении `✓` → `POST /profile/phone/request-code { new_phone }`.
- После успешной отправки кода – открывается шаг подтверждения с полем OTP (segmented, 4 цифры) и таймером повторной отправки (30 с).
- Кнопка «Отменить» – возврат в просмотр без изменений.
- При подтверждении → `POST /profile/phone/confirm { new_phone, code }`. При успехе – снек «Изменения сохранены» и перерисовка с новым номером. При ошибке – сообщение под полем.

### 3.5. Выход из аккаунта
- Кнопка «Выйти» открывает модальное окно (мини-шторку) с предупреждением: «Чтобы снова записаться на класс, нужно будет войти по номеру телефона.»
- Кнопка «Отмена» – закрыть, остаться в профиле.
- Кнопка «Выйти» → `POST /auth/logout`. При успехе (204) токены очищаются (`setTokens(null, null)`), модалка закрывается, происходит переход на экран входа.
- При ошибке сети – снек «Не удалось выйти. Проверьте соединение и попробуйте снова.», пользователь остаётся в профиле.

### 3.6. Удаление аккаунта
- Кнопка «Удалить аккаунт» открывает модальное окно с предупреждением о необратимости и последствиях (активные брони аннулируются, прошедшие анонимизируются).
- Кнопка «Отмена» – закрыть.
- Кнопка «Удалить аккаунт» → `DELETE /profile`. При успехе (204) токены очищаются, модалка закрывается, происходит переход на экран входа, где отображается снек «Аккаунт удалён».
- При ошибке – снек «Не удалось удалить аккаунт».

## 5. Ручная проверка (чек-лист)
- [ ] Перейти на вкладку «Профиль» – отображаются имя и телефон (замаскированный), кнопки «Выйти» и «Удалить аккаунт».
- [ ] Нажать карандаш у имени – поле становится редактируемым.
- [ ] Изменить имя, нажать `✓` – снек «Профиль обновлён», поле возвращено в просмотр с новым именем.
- [ ] Нажать карандаш у телефона, изменить номер, нажать `✓` – открыт шаг ввода кода.
- [ ] Ввести неверный код – ошибка под полем, кнопка «Подтвердить» возвращается в активное состояние.
- [ ] Нажать «Отправить код повторно» – таймер 0:30, по истечении ссылка активна.
- [ ] Ввести верный код – снек «Изменения сохранены», профиль обновлён с новым номером.
- [ ] Нажать «Выйти» – открывается модалка, нажать «Отмена» – модалка закрыта, пользователь в профиле.
- [ ] Нажать «Выйти» → «Выйти» – сессия завершена, открыт экран входа.
- [ ] Нажать «Удалить аккаунт» – открывается модалка с предупреждением.
- [ ] Нажать «Отмена» – аккаунт не удалён.
- [ ] Нажать «Удалить аккаунт» → «Удалить аккаунт» – аккаунт удалён, переход на экран входа, снек «Аккаунт удалён».
- [ ] «Правила студии» и «Поддержка» – показывают информационный снек (временно).
- [ ] Проверить, что при ошибке загрузки профиля (5xx) отображается заглушка с кнопкой «Обновить», а кнопка «Выйти» остаётся доступной.
