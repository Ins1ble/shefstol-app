# План реализации Web-клиента для «Шеф-стол»

## Контекст анализа
Документ проектирует клиентское веб-приложение «Шеф-стол», адаптированное под мобильные устройства. Основа: одностраничное приложение (SPA) на HTML5, CSS3, JavaScript (ES6) с использованием Bootstrap 5 для адаптивной вёрстки. Реализована чистая модульная архитектура с разделением на экраны и переиспользуемые сервисы.

Источники:
- `01-analysis/2-requirements/` – функциональные и нефункциональные требования
- `01-analysis/3-design-brief/` – экранные требования SCR-001..007, BS-001..004
- `01-analysis/5-mobile-app-spec/` – детальное ТЗ экранов и логик
- `01-analysis/api/` – контракты API (Auth, Profile, Slots, Bookings, Chefs)

## Выводы по требованиям
MVP-клиент покрывает только роль клиента:
- SCR-001 вход/регистрация по телефону и OTP
- SCR-002 список слотов с фильтрами BS-001
- SCR-003 карточка слота с адресом BS-004
- SCR-004 оформление брони
- BS-002 успех бронирования и запрос push-разрешения
- SCR-005 мои бронирования
- SCR-006 детали брони и отмена BS-003
- SCR-007 профиль, выход, операции профиля

Сквозные логики, реализованные как чистые функции:
- LOGIC-001 OTP-авторизация и сессия (автообновление токена)
- LOGIC-002 доступность: max_seats = min(free_seats, capacity_cap, 3) и rental_count ≤ free_rental_kits
- LOGIC-003 цена: price × seats + rental_price × rental
- LOGIC-004 отмена: правило 2 часов
- LOGIC-005 фильтры слотов: OR внутри группы, AND между группами
- LOGIC-006 адрес студии: текст + статическая карта при координатах
- LOGIC-007 push-разрешение после первой успешной брони
- LOGIC-008 паттерн состояний экрана (скелетоны, снек-бар)

## Архитектура
- **Экранная модель:** каждый экран рендерится в соответствующий `<div>` контейнер, переключение через `showScreen(name)`.
- **Навигация:** таб-бар «Классы», «Мои записи», «Профиль» для авторизованной зоны; вложенные экраны скрывают таб-бар.
- **Управление состоянием:** глобальный объект `state`, локальные состояния в замыканиях.
- **API-сервис:** `api.js` с методами `apiGet`, `apiPost`, `apiPatch`, `apiDelete`. Автоматически добавляет Bearer-токен, при 401 выполняет тихий рефреш через `/auth/refresh`, при неудаче перенаправляет на экран входа.
- **Модальные окна:** Bootstrap Modal используются для BS-001 (фильтры), BS-003 (подтверждение отмены), BS-004 (адрес студии), а также для подтверждения выхода и удаления аккаунта в SCR-007.

## Ключевые паттерны
- **Состояния экранов (LOGIC‑008):** Loading → Content / Empty / Error. Загрузка отображается скелетонами, ошибки – снек-баром.
- **Мобильная адаптация:** крупные тач‑зоны (≥44pt), высокий контраст, viewport‑cover, таб‑бар внизу.
- **Безопасность:** токены в localStorage, автоматическое обновление сессии, очистка при выходе/удалении.

## Критерии приёмки
- Все экраны отображаются согласно дизайн‑брифингу.
- Авторизация работает без пароля, сессия сохраняется между перезагрузками.
- Бронирование с идемпотентностью, все ошибки обрабатываются.
- Отмена по правилу 2 часов, оба случая (ранняя/поздняя) работают.
- Профиль позволяет редактировать данные, менять телефон через OTP, выходить и удалять аккаунт.
- Интерфейс адаптирован для мобильных устройств.
