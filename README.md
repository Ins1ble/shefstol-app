
# Shefstol App — Кулинарная студия «Шеф-стол»

Веб-приложение для самостоятельной записи на кулинарные мастер-классы студии «Шеф-стол».  
Заменяет ручную запись через WhatsApp и Google-таблицы: клиенты просматривают расписание, бронируют места, выбирают инвентарь и отменяют записи.

Проект выполнен в рамках **Летней школы SurfStudio 2026**.  
Выбранный бриф: **«Кулинарная студия»**.

---

## Цель проекта

Повторить полный цикл разработки продукта по готовой аналитике:

1. Сформулировать требования к MVP на основе брифа заказчика.
2. Спроектировать архитектуру и модель данных.
3. Реализовать не менее 3 фиг.
4. Написать тест-кейсы, найти и исправить баги.
5. Задокументировать каждую фигу и баг.

---

## Ключевые роли и зоны ответственности

- **Клиент** — единственная роль, реализованная в MVP. Просматривает классы, записывается, отменяет брони, управляет профилем.
- **Шеф-повара и Владелец (Артём)** — работают через существующую инфраструктуру студии, в текущий скоуп не входят.

## Функциональные возможности (MVP)

### Экран входа / регистрации (SCR-001)

- Вход по номеру телефона с SMS-кодом (OTP).
- Трёхшаговый процесс: телефон → код → имя (для новых пользователей).
- Таймер повторной отправки кода, защита от подбора.
- Автоматическое сохранение сессии (JWT access + refresh).

### Список классов и фильтры (SCR-002, BS-001)

- Отображение ближайших 7 дней с группировкой по датам.
- Карточки с программой, шефом, ценой, количеством свободных мест.
- Заполненные классы помечаются «Мест нет» и не кликабельны.
- Фильтры: по дате (пресеты + произвольный диапазон), типу программы (новичковая/опытная), конкретному шефу, «только со свободными местами».
- Индикатор активных фильтров, сброс фильтров, пустые состояния.

### Карточка класса (SCR-003)

- Полная информация: программа, описание, длительность, шеф, адрес, места, прокатные наборы, цена.
- Статическая карта-превью (Яндекс.Карты) при наличии координат.
- Кнопка «Записаться» (неактивна при отсутствии мест).

### Оформление брони (SCR-004)

- Выбор числа мест (1–3) с ограничением `min(free_seats, capacity_cap, 3)`.
- Для каждого места – переключатель «свой инвентарь» / «прокатный набор».
- Динамический расчёт итоговой стоимости с учётом тарифа проката.
- Поле «Пищевые ограничения» (опционально).
- Идемпотентное создание брони (`Idempotency-Key`), защита от двойного бронирования.
- Обработка ошибок: нехватка мест (E1), нехватка проката (E2), гонка запросов (E3), сетевой сбой (E4).

### Подтверждение записи (BS-002)

- Сводка брони: дата, программа, шеф, места, инвентарь, итоговая стоимость.
- Кнопки «Мои записи», «Добавить в календарь» (открывает Google Календарь), «Готово».
- Запрос системного разрешения на push-уведомления после первой брони.

### Мои бронирования (SCR-005)

- Автоматическое разделение на «Предстоящие» и «Историю».
- Карточки со статусом (активна, отменена, поздняя отмена, отменена студией).
- Переход к деталям брони.

### Детали брони и отмена (SCR-006, BS-003)

- Полная информация о брони, включая дедлайн бесплатной отмены.
- Кнопка «Отменить запись» (активна только для будущих активных броней).
- Модальное окно с объяснением правила 2 часов (ранняя/поздняя отмена).
- После отмены статус обновляется автоматически.

### Профиль клиента (SCR-007)

- Отображение имени и маскированного телефона.
- Инлайн-редактирование имени и смена телефона (с подтверждением через OTP).
- Модальные окна подтверждения выхода и удаления аккаунта.
- При удалении аккаунта активные брони аннулируются, прошедшие анонимизируются.

### Адрес студии (BS-004)

- Модальное окно с полным адресом, статической картой (при координатах) и ссылкой «Открыть в Яндекс.Картах».

## Запуск приложения

1. **Установите зависимости** (желательно в виртуальном окружении):

   ```bash
   pip install flask flask-cors pytest
   ```

2. **Запустите сервер**:

   ```bash
   cd 02-development/backend
   python app.py
   ```

   Сервер будет доступен по адресу `http://127.0.0.1:5000`.

3. **Откройте браузер** и перейдите по адресу `http://127.0.0.1:5000`.

   Начните с ввода номера телефона (например, `+79991234567`), получите код (он выводится в консоли сервера) и войдите в приложение.

4. **Запуск тестов** (из корня репозитория):

   ```bash
   pytest 03-test/unit/ -v
   ```

   Ожидаемый результат: **25 тестов пройдено**.
