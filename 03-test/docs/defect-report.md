## BUG‑001 — Зацикливание входа после верификации кода

**Серьёзность:** Minor  
**Статус:** Исправлен  
**Дата обнаружения:** 2026-07-06  
**Окружение:** Chrome 126, Flask dev server, Windows 11  

### Описание
После успешной проверки OTP‑кода приложение возвращалось на экран ввода
телефона вместо перехода на главный экран.

### Шаги воспроизведения
1. Открыть приложение, ввести номер телефона, нажать «Получить код».
2. Ввести корректный код, нажать «Подтвердить».

### Фактический результат
Снова отображается поле ввода телефона (шаг 1), войти невозможно.

### Ожидаемый результат
Приложение переходит на главный экран со списком классов.

### Причина
В файле `auth.js` после получения ответа от `/auth/verify-code`
отсутствовал вызов `setTokens(access_token, refresh_token)`.
Из‑за этого модуль `api.js` не получал токен, все защищённые запросы
возвращали 401, и интерцептор перенаправлял на экран входа.

### Исправление
Добавлена строка `setTokens(data.tokens.access_token, data.tokens.refresh_token)`
сразу после успешного ответа `verifyAuthCode`.

### Проверка исправления
Повторное тестирование по тест‑кейсам TC‑AUTH‑001, TC‑AUTH‑004, TC‑AUTH‑005
показало корректную работу входа.
